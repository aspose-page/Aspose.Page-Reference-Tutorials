## O que é um documento XPS e por que criá‑lo?

XPS (XML Paper Specification) é um formato de documento de layout fixo desenvolvido pela Microsoft. Ele preserva layout, fontes e gráficos em diferentes plataformas, tornando‑o ideal para faturas, relatórios e formulários imprimíveis. Criar documentos XPS programaticamente oferece controle total sobre design e automação, eliminando a necessidade de autoria manual.

## Por que usar Aspose.Page para .NET para criar documentos XPS?

- **Sem dependências externas** – funciona pronto para uso em qualquer projeto .NET.  
- **Alta fidelidade** – arquivos XPS gerados são renderizados de forma idêntica no Windows, Linux e plataformas móveis.  
- **Suporte a múltiplos formatos** – a mesma base de código também pode gerar arquivos PostScript, permitindo fluxos de trabalho “escreva uma vez, gere muitos”.

## Perguntas Frequentes

**Q: Como iniciar um novo documento XPS do zero?**  
A: Instancie a classe `Document`, adicione uma `Page`, então use objetos `Graphics` para desenhar texto, imagens ou formas.

**Q: Posso converter um PDF existente para XPS usando Aspose.Page?**  
A: A conversão direta de PDF‑para‑XPS é tratada pelo Aspose.PDF, mas você pode exportar o conteúdo do PDF para imagens e incorporá‑las em um documento XPS com Aspose.Page.

**Q: É possível editar um arquivo XPS existente sem recriá‑lo?**  
A: Sim – carregue o arquivo com `Document.Load`, modifique páginas ou adicione novo conteúdo, então salve novamente.

**Q: Qual a melhor maneira de gerar um arquivo PostScript para impressão?**  
A: Use a mesma API `Document`, mas chame `Save` com a opção `SaveFormat.PostScript`. Isso garante saída pronta para impressão.

**Q: Existem limites de tamanho para arquivos XPS ou PostScript?**  
A: A biblioteca lida eficientemente com arquivos grandes; porém, para documentos extremamente extensos, considere transmitir o conteúdo em streaming e usar `Document.OptimizeResources()`.
