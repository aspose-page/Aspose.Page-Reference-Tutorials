# Aspose.Page Java Tutorial – установка пользовательского размера страницы при добавлении страниц в PostScript

## Введение
В современных Java‑приложениях часто требуется **установка пользовательского размера страницы** для вывода в PostScript — будь то генерация счетов, билетов или пользовательской графики. В этом руководстве вы узнаете, как **установить пользовательский размер страницы** для каждой страницы, добавить несколько страниц и в конце **создать файл PostScript**, соответствующий вашим точным требованиям к разметке. Мы пройдём по коду шаг за шагом, чтобы вы могли быстро применить эту технику в своих проектах.

## Быстрые ответы
- **Могу ли я установить разные размеры страниц для каждой страницы?** Да, вы можете открывать страницы с пользовательскими размерами, используя `document.openPage(width, height)`.  
- **Нужна ли лицензия для использования в продакшене?** Для не‑оценочных развертываний требуется действующая лицензия Aspose.Page.  
- **Какие версии Java поддерживаются?** Библиотека работает с Java 8 и новее.  
- **Является ли API потокобезопасным?** Экземпляры Document не являются потокобезопасными; создавайте отдельный `PsDocument` для каждого потока.  
- **Насколько большим может быть файл PostScript?** Aspose.Page эффективно обрабатывает многомегабайтные файлы; использование памяти масштабируется с содержимым, а не с количеством страниц.  
- **Могу ли я использовать перегрузку open page width/height?** Абсолютно — `openPage(double width, double height)` позволяет указать любые размеры в пунктах.  

## Предварительные требования
- Базовое понимание программирования на Java.  
- Aspose.Page for Java, добавленный в ваш проект (Maven/Gradle или вручную JAR).  
- Среда разработки Java (IDE, JDK 8+).  

## Импорт пакетов
Для начала импортируйте необходимые классы из библиотеки Aspose.Page.

```java
import java.io.FileOutputStream;
import com.aspose.eps.PsDocument;
import com.aspose.eps.device.PsSaveOptions;
```

## Шаг 1: Создание многостраничного документа PostScript
Сначала создайте новый `PsDocument`, настроенный для нескольких страниц. Это устанавливает поток вывода и сообщает библиотеке, что мы будем работать с многостраничным файлом.

```java
// The path to the documents directory.
String dataDir = "Your Document Directory";
// Create output stream for PostScript document
FileOutputStream outPsStream = new FileOutputStream(dataDir + "AddPages2_outPS.ps");
// Create save options with A4 size
PsSaveOptions options = new PsSaveOptions();
// Set variable that indicates if resulting PostScript document will be multipaged
boolean multiPaged = true;
// Create new multipaged PS Document with one page opened
PsDocument document = new PsDocument(outPsStream, options, multiPaged);
```

## Шаг 2: Добавление содержимого на первую страницу
Когда документ готов, вы можете добавить любое необходимое содержимое на первую страницу. После завершения закройте страницу, чтобы зафиксировать её содержимое.

```java
// Add content to the first page
// Close the first page
document.closePage();
```

## Как установить пользовательский размер страницы
Если размер страницы по умолчанию не подходит, вы можете **установить пользовательский размер страницы** при открытии новой страницы. Это полезно для чеков, этикеток или любой нестандартной разметки.

## Шаг 3: Добавление второй страницы с другим размером
Ниже мы открываем вторую страницу и явно указываем пользовательскую ширину и высоту (в пунктах). Это демонстрирует, как установить пользовательский размер страницы для отдельных страниц, предоставляя возможность работать с **разными размерами страниц** в одном документе.

```java
// Add the second page with a different size
document.openPage(500, 300);
// Add content to the second page
// Close the second page
document.closePage();
```

## Шаг 4: Сохранение документа
Наконец, зафиксируйте изменения, сохранив документ. Все страницы — включая те, что имеют пользовательские размеры — записываются в выходной файл.

```java
// Save the document
document.save();
```

Следуя этим шагам, вы сможете без проблем добавлять страницы и **устанавливать пользовательский размер страницы** в Java‑документе PostScript с помощью Aspose.Page, получая полный контроль над разметкой каждой страницы.

## Почему стоит использовать Aspose.Page для установки пользовательского размера страницы?
- **Точность:** Размеры задаются в пунктах, поэтому вы получаете точный контроль над шириной и высотой страницы.  
- **Гибкость:** Сочетайте **разные размеры страниц** в одном файле PostScript.  
- **Производительность:** Библиотека потоково передаёт содержимое напрямую в файл вывода, что делает её подходящей для масштабных сценариев **генерации файлов PostScript**.  
- **Богатый API:** Поддерживает рисование графики, встраивание изображений и добавление текста — всё это с учётом заданных вами пользовательских размеров.  

## Распространённые проблемы и решения
| Issue | Solution |
|-------|----------|
| **Размеры страницы выглядят перепутанными** | Помните, что `openPage(width, height)` ожидает сначала ширину, затем высоту (обе в пунктах). |
| **Содержимое выходит за пределы страницы** | Используйте систему координат `PsGraphics` для размещения элементов внутри пользовательских границ или масштабируйте ваш рисунок. |
| **Ошибки out‑of‑memory при работе с огромными документами** | Включите потоковую передачу, записывая напрямую в `FileOutputStream`, как показано, и избегайте одновременной загрузки больших изображений в память. |

## Часто задаваемые вопросы
### Могу ли я добавить страницы разных размеров в один документ PostScript?
Да, как показано в этом руководстве, вы можете добавлять страницы с разными размерами в многостраничный документ PostScript.  

### Есть ли ограничения на количество страниц, которые я могу добавить?
Aspose.Page поддерживает добавление практически неограниченного количества страниц в документ.  

### Могу ли я добавить пользовательское содержимое, например изображения или графику, на страницы?
Абсолютно! Aspose.Page позволяет добавлять широкий спектр содержимого, включая текст, изображения и другие графические элементы.  

### Подходит ли Aspose.Page для работы с большими документами?
Да, Aspose.Page разработан для эффективной работы как с небольшими, так и с большими документами.

**Additional Q&A**

**Q:** *Какие форматы изображений поддерживаются при рисовании на странице PostScript?*  
**A:** Вы можете напрямую встраивать изображения PNG, JPEG, BMP и GIF, используя API рисования.  

**Q:** *Как изменить DPI по умолчанию для документа?*  
**A:** Установите `PsSaveOptions.setResolution(int dpi)` перед созданием `PsDocument`.  

**Q:** *Могу ли я зашифровать файл PostScript паролем?*  
**A:** Сам PostScript не поддерживает шифрование, но вы можете обернуть вывод в PDF и применить настройки безопасности при необходимости.
