## Как изменить размер EPS с помощью Aspose.Page в Java
Изменение размера EPS‑файлов часто является первым шагом, когда нужно вписать графику в заранее заданный макет или уменьшить размер файла для более быстрой передачи. Ниже представлена краткая пошаговая инструкция. Фактический фрагмент кода доступен в отдельном учебнике по изменению размера, ссылка на который указана ниже.

1. **Load the EPS document** – Используйте класс `Page` для открытия исходного файла.  
2. **Define the new dimensions** – Установите требуемую ширину и высоту в пунктах (1 pt = 1/72 дюйма).  
3. **Apply the resize operation** – Вызовите `page.resize()` или измените размер страницы через `PageInfo`.  
4. **Save the result** – Экспортируйте в EPS, PDF, PNG или любой другой поддерживаемый формат.

> **Pro tip:** Когда нужен лишь иной размер для печати, оставьте оригинальный EPS нетронутым и создайте временную копию с изменёнными размерами. Это сохраняет мастер‑файл для будущего повторного использования.

## Часто задаваемые вопросы
**Q: Можно ли обрабатывать несколько EPS‑файлов пакетно?**  
A: Да. Оберните логику обрезки или изменения размера в цикл и переиспользуйте один объект `Page` для каждого файла.

**Q: Поддерживает ли Aspose.Page EPS с встроенными растровыми изображениями?**  
A: Абсолютно. Как векторные, так и растровые элементы сохраняются при манипуляции.

**Q: Можно ли после редактирования конвертировать EPS в другие форматы (PDF, PNG)?**  
A: Да. После обрезки или изменения размера вы можете вызвать `page.save("output.pdf", SaveFormat.Pdf)` или `SaveFormat.Png`.

**Q: Что делать, если EPS‑файл повреждён?**  
A: API бросает `FileFormatException`. Перехватите исключение и проверьте исходный файл перед обработкой.

**Q: Нужно ли закрывать какие‑либо ресурсы вручную?**  
A: Класс `Page` реализует `Closeable`; используйте блок try‑with‑resources или вызовите `page.close()` для освобождения памяти.
