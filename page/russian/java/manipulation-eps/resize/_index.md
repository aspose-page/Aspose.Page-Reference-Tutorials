# Как изменить размер EPS‑файлов в Java с помощью Aspose.Page

## Введение
Если вы ищете надёжный способ **как изменить размер EPS** файлов программно, вы попали по адресу. В этом руководстве мы пройдем процесс изменения размеров EPS‑изображений в Java с использованием библиотеки Aspose.Page. Независимо от того, нужно ли вам удвоить размер, уменьшить его до конкретных измерений или работать с процентами, приведённые ниже шаги дают полный контроль над конечными размерами. Понимание **как изменить размер eps** необходимо, когда требуется адаптировать графику под разные макеты печати или разрешения экрана.

## Быстрые ответы
- **Какая библиотека нужна?** Aspose.Page for Java  
- **Можно ли изменять размер в пунктах, дюймах или миллиметрах?** Да — API поддерживает все три единицы измерения плюс проценты.  
- **Нужна ли лицензия для разработки?** Бесплатная пробная версия подходит для тестирования; для продакшна требуется лицензия.  
- **Какая версия Java требуется?** Java 8 или новее.  
- **Является ли код потокобезопасным?** Каждый экземпляр `PsDocument` изолирован, поэтому файлы можно обрабатывать параллельно.  

## Что такое EPS и зачем его менять размер?
Encapsulated PostScript (EPS) — популярный векторный графический формат, используемый в издательском деле и печати. Иногда исходный EPS‑файл создаётся в размере, не соответствующем требуемому результату — например, логотип, разработанный в 72 пт, может потребоваться в 144 пт для более крупного брошюра. Знание **как изменить размер eps** позволяет сохранять векторное качество, одновременно адаптируя размеры под любой рабочий процесс.

## Почему стоит использовать Aspose.Page для изменения размера EPS?
- **Полный контроль над единицами** — пункты, дюймы, миллиметры или проценты.  
- **Без внешних зависимостей** — чистый Java API, без нативных библиотек.  
- **Высокая производительность** — подходит для пакетной обработки на серверах.  
- **Сохраняет векторную точность** — результат остаётся масштабируемым без растеризации.

## Импорт пакетов
В вашем Java‑проекте добавьте необходимые импорты, чтобы работать с объектами Aspose.Page и стандартными потоками ввода‑вывода.

```java
import java.awt.Dimension;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import com.aspose.eps.PsDocument;
import com.aspose.page.DimensionF;
import com.aspose.page.Units;
```

## Как изменить размеры EPS с разными единицами измерения
Библиотека позволяет **изменять размеры eps** просто выбирая соответствующее значение перечисления `Units`. Ниже мы пройдем один и тот же пятишаговый шаблон для пунктов, дюймов, миллиметров и процентов. Единственное, что меняется — единица, которую вы передаёте в `resizeEps`.

## Как изменить размер EPS, используя пункты
Ниже приведён полный пошаговый пример, удваивающий размер EPS‑файла, используя пункты в качестве единицы измерения.

### Шаг 1: Настройте входной поток
```java
FileInputStream inputEpsStream = new FileInputStream(dataDir + "input.eps");
```

### Шаг 2: Инициализируйте объект `PsDocument`
```java
PsDocument doc = new PsDocument(inputEpsStream);
```

### Шаг 3: Получите текущий размер EPS‑изображения
```java
Dimension oldSize = doc.extractEpsSize();
```

### Шаг 4: Создайте выходной поток для изменённого файла
```java
FileOutputStream outputEpsStream = new FileOutputStream(dataDir + "output_resize_points.eps");
```

### Шаг 5: Измените размер и сохраните EPS, используя пункты
```java
doc.resizeEps(outputEpsStream, new Dimension2D.Double(oldSize.width * 2, oldSize.height * 2), Units.Points);
```

## Как изменить размер EPS, используя дюймы
Тот же пятишаговый шаблон применяется; просто замените `Units.Points` на `Units.Inches` и скорректируйте коэффициент масштабирования до нужного значения в дюймах.

## Как изменить размер EPS, используя миллиметры
Опять же, следуйте тем же шагам, заменив единицу на `Units.Millimeters`. Это удобно, когда требуются метрические размеры для печатных процессов.

## Как изменить размер EPS, используя проценты
Для масштабирования в процентах оставьте единицу `Units.Percent`. Умножьте исходную ширину и высоту на требуемый процент (например, `0.5` для уменьшения на 50 %).

## Распространённые ошибки и советы
- **Всегда закрывайте потоки** — в продакшн‑коде оборачивайте потоки в `try‑with‑resources`, чтобы избежать блокировок файлов.  
- **Сохраняйте соотношение сторон** — умножайте ширину и высоту на один и тот же коэффициент, если только не требуется искажать изображение.  
- **Проверьте DPI** — изменение размера не меняет DPI; если нужен иной DPI, скорректируйте его отдельно после изменения размеров.  
- **Потокобезопасность** — создавайте новый `PsDocument` для каждого потока; совместное использование одного экземпляра может привести к неожиданным результатам.
