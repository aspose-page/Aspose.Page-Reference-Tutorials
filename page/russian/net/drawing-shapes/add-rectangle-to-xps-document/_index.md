# Создание XPS‑документа: Добавление прямоугольника с помощью Aspose.Page для .NET

## Введение

В этом руководстве вы **создадите XPS‑документ** и нарисуете в нём прямоугольник, используя библиотеку Aspose.Page для .NET. Добавление фигур, таких как прямоугольники, часто требуется при программной генерации счетов, отчетов или пользовательской графики. Следуйте инструкциям ниже, и уже через несколько минут у вас будет готовый XPS‑файл.

## Быстрые ответы
- **Что я могу сгенерировать?** XPS‑документы с векторной графикой и текстом.  
- **Какая библиотека?** Aspose.Page для .NET (доступна бесплатная trial‑версия).  
- **Сколько времени займет?** Около 5‑10 минут на реализацию и запуск.  
- **Нужна ли лицензия?** Для использования в продакшене требуется временная лицензия.  
- **Можно ли сохранить результат как XPS‑файл?** Да – метод `Save` записывает **сохранённый XPS‑файл** на диск.

## Что значит «создать XPS‑документ»?

Создание XPS‑документа подразумевает программную сборку описания страницы в формате XML Paper Specification. Полученный файл независим от разрешения, что делает его идеальным для печати и высококачественного отображения на разных платформах.

## Почему стоит использовать Aspose.Page для создания XPS‑документа?

- **Полная поддержка .NET** – работает с .NET Framework, .NET Core и .NET 5/6+.  
- **Богатый API рисования** – включает геометрию путей, кисти, пера и работу с текстом.  
- **Отсутствие внешних зависимостей** – всё выполняется в процессе без необходимости установки Office или Acrobat.

## Импорт пространств имён

В вашем .NET‑проекте импортируйте необходимые пространства имён:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

## Шаг 1: Установить каталог документа

```csharp
// ExStart:3
// The path to the documents directory.
string dataDir = "Your Document Directory";
// ExEnd:3
```

> **Полезный совет:** используйте абсолютный путь или `Path.Combine`, чтобы избежать проблем с разделителями пути на разных ОС.

## Шаг 2: Создать новый XPS‑документ

```csharp
// ExStart:4
// Create new XPS Document
XpsDocument doc = new XpsDocument();
// ExEnd:4
```

На данном этапе вы **создали объект XPS‑документа**, который будет содержать все элементы рисования.

## Шаг 3: Добавить прямоугольник (с помощью create path geometry)

```csharp
// ExStart:5
// CMYK (blue) solid color stroked rectangle in the lower left
XpsPath path = doc.AddPath(doc.CreatePathGeometry("M 20,10 L 220,10 220,100 20,100 Z"));
path.Stroke = doc.CreateSolidColorBrush(
    doc.CreateColor(dataDir + "uswebuncoated.icc", 1.0f, 1.000f, 0.000f, 0.000f, 0.000f));
path.StrokeThickness = 12f;
// ExEnd:5
```

Вызов `CreatePathGeometry` **создаёт геометрию пути**, определяющую контур прямоугольника. Вы можете изменить строку команды, похожую на SVG, чтобы рисовать другие фигуры.

## Шаг 4: Сохранить документ (сохранить XPS‑файл)

```csharp
// ExStart:6
// Save resultant XPS document
doc.Save(dataDir + "AddRectangleXPS_out.xps");
// ExEnd:6
```

Вызов `Save` записывает **сохранённый XPS‑файл** в указанное вами место.

Поздравляем! Вы успешно **создали XPS‑документ**, добавили прямоугольник и сохранили файл.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| `FileNotFoundException` при `doc.Save` | `dataDir` указывает на несуществующую папку | Убедитесь, что каталог существует, или создайте его с помощью `Directory.CreateDirectory(dataDir)`. |
| Прямоугольник не виден | Отсутствует профиль цвета линии или геометрия пути сформирована неверно | Проверьте путь к ICC‑файлу и строку геометрии (`M … Z`). |
| Замедление работы при больших документах | Слишком много отдельных вызовов `AddPath` | Группируйте операции рисования или переиспользуйте объекты кистей/пен. |
