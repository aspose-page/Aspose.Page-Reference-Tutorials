# Конвертация XPS в PDF с помощью Aspose.Page для .NET

## Введение

В этом руководстве вы узнаете **как конвертировать XPS в PDF** с помощью библиотеки Aspose.Page для .NET. Конвертация XPS в PDF является распространённой задачей, когда необходимо поделиться XPS‑документами с пользователями, у которых только PDF‑просмотрщики, или когда вы хотите встроить XPS‑контент в более крупные PDF‑рабочие процессы. Мы пройдём каждый шаг, объясним, почему каждое параметр важен, и покажем, как точно настроить вывод — например, установить качество JPEG и применить сжатие изображений PDF.

## Краткие ответы
- **Какая библиотека лучше всего подходит для конвертации XPS в PDF?** Aspose.Page for .NET
- **Нужна ли лицензия для продакшн?** Да, требуется коммерческая лицензия; доступна бесплатная пробная версия.
- **Можно ли контролировать качество изображения?** Конечно — используйте `JpegQualityLevel` и `PdfImageCompression`.
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.
- **Можно ли конвертировать несколько XPS‑файлов в один PDF?** Да, путем перебора файлов и объединения результатов.

## Требования

Прежде чем начать процесс конвертации, убедитесь, что у вас есть следующие требования:

- **Aspose.Page for .NET Library** – Убедитесь, что библиотека Aspose.Page for .NET установлена в вашей среде разработки. Вы можете скачать её из [Aspose.Page documentation](https://reference.aspose.com/page/net/).
- **Development Environment** – Настройте .NET среду разработки с Visual Studio или любой другой совместимой IDE.
- **XPS Document** – Подготовьте XPS‑документ, который вы хотите конвертировать в PDF. Это может быть ваш пример XPS‑файла, хранящийся в указанной директории.

## Импорт пространств имён

Прежде чем погрузиться в код, импортируем необходимое пространство имён, чтобы функции Aspose.Page для .NET были доступны в нашем проекте:

```csharp
using Aspose.Page.XPS;
```

## Пошаговое руководство

### Шаг 1: Инициализация директории документа

Укажите папку, в которой находится ваш исходный XPS‑файл и куда будет сохранён полученный PDF.

```csharp
string dataDir = "Your Document Directory";
```

Замените `"Your Document Directory"` на абсолютный или относительный путь к папке, содержащей ваш XPS‑документ.

### Шаг 2: Открытие потоков для вывода PDF и ввода XPS

Мы используем два файловых потока — один для чтения XPS‑файла и другой для записи сгенерированного PDF.

```csharp
using (System.IO.Stream pdfStream = System.IO.File.Open(dataDir + "XPStoPDF_out.pdf", System.IO.FileMode.OpenOrCreate, System.IO.FileAccess.Write))
using (System.IO.Stream xpsStream = System.IO.File.Open(dataDir + "input.xps", System.IO.FileMode.Open))
```

> **Pro tip:** Убедитесь, что пути указаны правильно и приложение имеет права чтения/записи в целевой папке.

### Шаг 3: Загрузка XPS‑документа

Создайте экземпляр `XpsDocument` из XPS‑потока. Объект `XpsLoadOptions` позволяет задать параметры загрузки, но значение по умолчанию подходит для большинства сценариев.

```csharp
XpsDocument document = new XpsDocument(xpsStream, new XpsLoadOptions());
```

### Шаг 4: Настройка параметров сохранения PDF

Здесь мы задаём параметры вывода PDF. Обратите внимание на использование **сжатия изображений PDF** (`PdfImageCompression.Jpeg`) и **качества JPEG** (`JpegQualityLevel = 100`). Эти настройки напрямую влияют на размер файла и визуальное качество.

```csharp
PdfSaveOptions options = new PdfSaveOptions()
{
    JpegQualityLevel = 100,
    ImageCompression = PdfImageCompression.Jpeg,
    TextCompression = PdfTextCompression.Flate,
    PageNumbers = new int[] { 1, 2, 6 }
};
```

- **`JpegQualityLevel`** – Управляет качеством JPEG‑изображений, встроенных в PDF (чем выше, тем лучше качество, но больше размер файла).
- **`ImageCompression`** – Выбирает алгоритм сжатия; JPEG идеален для фотографических изображений.
- **`TextCompression`** – Сжатие Flate уменьшает размер PDF без потери качества текста.
- **`PageNumbers`** – Позволяет **сохранять XPS в PDF** только для выбранных страниц.

### Шаг 5: Создание устройства рендеринга PDF

`PdfDevice` выступает в роли цели рендеринга, записывая данные PDF в поток, открытый ранее.

```csharp
PdfDevice device = new PdfDevice(pdfStream);
```

### Шаг 6: Сохранение документа в PDF

Наконец, вызовите метод `Save`, передав устройство рендеринга и настроенные параметры.

```csharp
document.Save(device, options);
```

После выполнения кода файл `XPStoPDF_out.pdf` появится в указанной директории, содержащий конвертированные страницы с применёнными настройками сжатия и качества.

## Почему стоит конвертировать XPS в PDF?

- **Универсальная доступность** – Просмотрщики PDF установлены практически на всех устройствах, тогда как XPS‑ридеры редки.
- **Сохранение макета** – PDF сохраняет точный визуальный макет, шрифты и графику оригинального XPS.
- **Дальнейшая обработка** – После преобразования в PDF вы можете объединять, аннотировать или цифрово подписывать документ с помощью других библиотек Aspose.

## Распространённые сценарии использования

- **Корпоративная отчётность** – Генерировать XPS‑отчёты из устаревших систем и конвертировать их в PDF для распространения.
- **Архивирование** – Хранить документы в формате PDF для долгосрочного сохранения, при этом иметь возможность создавать их из XPS‑источников.
- **Веб‑сервисы** – Предоставлять API‑конечную точку, принимающую загрузки XPS и мгновенно возвращающую PDF‑файлы.

## Устранение неполадок и советы

- **Файл не найден** – Проверьте путь `dataDir` и убедитесь, что имя XPS‑файла точно совпадает.
- **Ошибки доступа** – Запустите Visual Studio от имени администратора или предоставьте права записи в папку вывода.
- **Большие PDF** – Если полученный PDF слишком велик, уменьшите `JpegQualityLevel` или переключите `ImageCompression` на `PdfImageCompression.Zip`.

## Часто задаваемые вопросы (AI‑Friendly)

**Q: Как установить качество JPEG при конвертации XPS в PDF?**  
A: Используйте свойство `JpegQualityLevel` внутри `PdfSaveOptions`. Установка значения 100 обеспечивает максимальное качество.

**Q: Что означает «pdf image compression» в данном контексте?**  
A: Это относится к параметру `ImageCompression`, который определяет, как изображения сжимаются внутри PDF (например, JPEG, Zip).

**Q: Могу ли я программно генерировать PDF без исходного XPS?**  
A: Да, Aspose.Page также поддерживает **c# generate pdf** напрямую из команд рисования, но это выходит за рамки данного руководства.

**Q: Есть ли способ конвертировать XPS в PDF без потери векторной графики?**  
A: Конверсия сохраняет векторные данные; просто избегайте растеризации изображений, оставляя `ImageCompression` установленным на JPEG или Zip по необходимости.

**Q: Поддерживает ли библиотека .NET Core?**  
A: Конечно – Aspose.Page for .NET работает с .NET Core, .NET 5, .NET 6 и более поздними версиями.
