# Изменение XPS‑документа с помощью Aspose.Page для .NET

## Введение

Добро пожаловать в наш пошаговый учебник о **как изменить xps документ** с помощью Aspose.Page для .NET. Нужно добавить подпись, водяной знак или просто разместить пользовательский текст на странице — этот учебник покажет, **как добавить текст** в XPS‑документ за несколько минут. Мы пройдемся по каждой строке кода, объясним, почему каждый шаг важен, и дадим советы, как избежать распространённых ошибок.

### Быстрые ответы
- **Что охватывает этот учебник?** Добавление текста подписи ("Confirmed") к выбранным страницам XPS‑файла.  
- **Какая библиотека требуется?** Aspose.Page for .NET (последняя версия).  
- **Нужна ли лицензия?** Временная лицензия подходит для тестирования; полная лицензия требуется для продакшна.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **Сколько времени занимает реализация?** Около 10 минут для базового добавления подписи.

## Что такое изменение XPS‑документа?

XPS (XML Paper Specification) — фиксированный формат документов Microsoft, аналогичный PDF. Изменение XPS‑документа означает программное изменение его визуального содержимого — добавление текста, изображений или фигур — без конвертации файла в другой формат. Aspose.Page предоставляет богатую объектную модель, позволяющую редактировать XPS‑файлы напрямую из вашего .NET‑кода.

## Почему использовать Aspose.Page для изменения XPS‑документов?

* **Полный контроль** – работа со страницами, глифами, кистями и трансформациями на низком уровне.  
* **Отсутствие внешних зависимостей** – чистая .NET‑библиотека, не требуется Office или компоненты Adobe.  
* **Кросс‑платформенность** – работает на Windows, Linux и macOS через .NET Core.  
* **Надёжная производительность** – эффективно обрабатывает большие документы и поддерживает продвинутую типографику.

## Требования

- **Aspose.Page for .NET** – Установите пакет NuGet или скачайте библиотеку из официальной документации **[here](https://reference.aspose.com/page/net/)**.  
- **Входной XPS‑файл** – Получите пример XPS‑документа (например, `input1.xps`) со **[страницы релизов Aspose](https://releases.aspose.com/page/net/)**.  
- **Рабочий каталог** – Создайте папку на компьютере для хранения входных и выходных файлов и запомните её полный путь; вы присвоите этот путь переменной `dir` в коде.  
- **Среда разработки** – Visual Studio 2019/2022, .NET Framework 4.7.2 или новее, либо любой проект .NET Core/5/6.

Теперь, когда всё готово, давайте погрузимся в код.

## Импорт пространств имён

В вашем .NET‑проекте начните с импорта необходимых пространств имён для Aspose.Page:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
using System.IO;
```

## Шаг 1: Открыть поток XPS‑документа

Мы откроем исходный XPS‑файл как поток и создадим объект `XpsDocument`, представляющий весь документ.

```csharp
// ExStart:3
// The path to the documents directory.
string dir = "Your Document Directory";
// Open a stream of XPS file
using (FileStream xpsStream = File.Open(dir + "input1.xps", FileMode.Open, FileAccess.Read))
{
    // Create PS document from stream
    XpsDocument document = new XpsDocument(xpsStream, new XpsLoadOptions());
    // Continue to the next step...
}
// ExEnd:3
```

*Подсказка:* Оберните поток в блок `using`, чтобы файловый дескриптор освобождался автоматически.

## Шаг 2: Создать текст подписи

Далее мы создаём кисть сплошного цвета, которая будет использоваться для отрисовки глифов подписи.

```csharp
// ExStart:4
// Create fill of the signature text
XpsSolidColorBrush textFill = document.CreateSolidColorBrush(Color.BlueViolet);
// Continue to the next step...
// ExEnd:4
```

Вы можете изменить `Color.BlueViolet` на любой `System.Drawing.Color`, соответствующий вашему бренду.

## Шаг 3: Определить страницы и добавить подпись

Мы укажем, какие страницы должны получить подпись, а затем добавим глифы на каждую страницу.

```csharp
// ExStart:5
// Define pages where signature will be set
int[] pageNumbers = new int[] {1, 2, 3};

// For every defined page set signature "Confirmed" at coordinates x=650 and y=950
for (int i = 0; i < pageNumbers.Length; i++)
{
    // Define active page
    document.SelectActivePage(pageNumbers[i]);

    // Create glyphs object
    XpsGlyphs glyphs = document.AddGlyphs("Arial", 24, FontStyle.Bold, 650, 900, "Confirmed");

    // Define fill for glyphs
    glyphs.Fill = textFill;
}
// Continue to the next step...
// ExEnd:5
```

*Почему такие координаты?* Значения X и Y измеряются в пунктах (1/72 дюйма). Отрегулируйте их, чтобы разместить текст точно там, где требуется в макете страницы.

## Шаг 4: Сохранить изменения в XPS‑документе

Наконец, запишите изменённый документ обратно на диск.

```csharp
// ExStart:6
// Save changed XPS document
document.Save(dir + "input1_out.xps");
// ExEnd:6
```

Новый файл `input1_out.xps` теперь содержит подпись «Confirmed» на страницах 1‑3.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| **Подпись не видна** | Неправильные координаты или страница не выбрана | Убедитесь, что `SelectActivePage` вызывается для каждой страницы, и скорректируйте значения X/Y. |
| **Исключение при `AddGlyphs`** | Шрифт не установлен на сервере | Убедитесь, что указанный шрифт (например, Arial) доступен, или внедрите пользовательский шрифт с помощью `document.AddFont`. |
| **Выходной файл повреждён** | Поток закрыт неправильно | Используйте конструкции `using` для всех потоков и при необходимости вызовите `document.Dispose()`. |
| **Снижение производительности на больших файлах** | Загрузка всего документа в память | Обрабатывайте страницы пакетами или используйте `XpsLoadOptions` с опциями потоковой загрузки (если доступно в новых версиях). |

О: Да, Aspose.Page регулярно обновляется для поддержки .NET Framework 4.5+, .NET Core 3.1+, .NET 5 и .NET 6.

**В: Можно ли настроить шрифт и стиль добавляемого текста?**
