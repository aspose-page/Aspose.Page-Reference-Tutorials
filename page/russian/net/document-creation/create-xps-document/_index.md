# Создание XPS‑документа с помощью Aspose.Page для .NET

## Введение

Добро пожаловать в наше пошаговое руководство по **how to create XPS document** с использованием Aspose.Page для .NET. В этом учебнике мы рассмотрим процесс генерации XPS‑файлов, широко используемого формата для электронных документов. Независимо от того, являетесь ли вы опытным разработчиком или только начинаете работать с Aspose.Page, это руководство поможет вам без труда создавать XPS‑документы с понятными примерами и подробными объяснениями.

## Быстрые ответы
- **Какая библиотека нужна?** Aspose.Page for .NET  
- **Можно ли запускать это на .NET Core?** Да, библиотека полностью поддерживает .NET Core и .NET 5/6  
- **Сколько строк кода?** Менее 20 строк для создания базового XPS‑файла  
- **Нужна ли лицензия для тестирования?** Доступна бесплатная пробная версия; лицензия требуется для продакшна  
- **В каком формате вывод?** XPS (XML Paper Specification)  

## Как создать XPS‑документ с помощью Aspose.Page для .NET

Ниже вы найдёте всё, что нужно перед началом кодирования, а затем краткое пошаговое руководство.

## Требования

Перед тем как приступить к учебнику, убедитесь, что у вас есть следующее:

1. **Aspose.Page for .NET Library:** Скачайте и установите библиотеку Aspose.Page с [download link](https://releases.aspose.com/page/net/).  
2. **Ваш каталог документов:** Выберите или создайте каталог в системе, куда вы будете сохранять полученные XPS‑файлы.

Теперь давайте перейдём к учебнику!

## Импорт пространств имён

Чтобы использовать Aspose.Page для .NET, необходимо импортировать требуемые пространства имён в ваш проект. Выполните следующие шаги:

### Шаг 1: Добавьте ссылку на Aspose.Page

В вашем проекте добавьте ссылку на библиотеку Aspose.Page for .NET. Необходимый DLL находится в загруженном пакете.

### Шаг 2: Импортируйте пространства имён

Включите следующие пространства имён в ваш файл кода:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

Теперь, когда мы настроили требования и импортировали необходимые пространства имён, разберём процесс создания XPS‑документа по нескольким шагам.

## Шаг 1: Установите каталог документа

```csharp
string dir = "Your Document Directory";
```

Замените `"Your Document Directory"` реальным путём, где вы хотите сохранить полученный XPS‑файл.

## Шаг 2: Создайте XPS‑документ

```csharp
XpsDocument xDocs = new XpsDocument();
```

Инициализируйте новый XPS‑документ, используя класс `XpsDocument`.

## Шаг 3: Добавьте глифы в документ

```csharp
var glyphs = xDocs.AddGlyphs("Arial", 12, FontStyle.Regular, 300f, 450f, "Hello World!");
```

Используйте метод `AddGlyphs` для добавления глифов (текста) в документ. При необходимости настройте шрифт, размер, стиль и позицию.

## Шаг 4: Установите цвет заливки глифов

```csharp
glyphs.Fill = xDocs.CreateSolidColorBrush(Color.Black);
```

Укажите цвет заливки для добавленных глифов. В этом примере используется чёрный, но вы можете выбрать любой цвет.

## Шаг 5: Сохраните результат

```csharp
xDocs.Save(dir + "output.xps");
```

Наконец, сохраните XPS‑документ в указанный каталог под желаемым именем файла. Полученный XPS‑файл будет содержать текст «Hello World!».

Поздравляем! Вы успешно создали XPS‑документ с помощью Aspose.Page для .NET.

## Общие советы и подводные камни

- **Путь к каталогу** – Используйте `Path.Combine(dir, "output.xps")`, чтобы избежать отсутствия разделителей пути на разных ОС.  
- **Наличие шрифта** – Шрифт, который вы указываете, должен быть установлен на машине, где выполняется код; иначе Aspose заменит его шрифтом по умолчанию.  
- **Несколько страниц** – Для создания многостраничных XPS‑файлов создавайте дополнительные объекты `XpsPage` и добавляйте контент на каждую страницу перед сохранением.
