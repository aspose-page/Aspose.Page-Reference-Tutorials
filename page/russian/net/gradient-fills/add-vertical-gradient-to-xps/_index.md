# Добавить вертикальный градиент в XPS с помощью Aspose.Page для .NET

## Введение

В этом руководстве вы **создадите XPS‑документ**, в котором будет плавный вертикальный градиент. Добавление градиентов — распространённый способ сделать XPS‑файлы более профессиональными, что идеально подходит для отчётов, брошюр или любой печатной графики. Мы пройдём каждый шаг, от настройки проекта до сохранения готового XPS‑файла, чтобы вы могли быстро применить линейный градиент к любой фигуре.

## Быстрые ответы
- **Что покрывает это руководство?** Добавление вертикального линейного градиента к пути в XPS‑документе.  
- **Какая библиотека требуется?** Aspose.Page для .NET.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для разработки; для продакшна требуется коммерческая лицензия.  
- **Сколько времени занимает реализация?** Около 10‑15 минут для базового примера.  
- **Можно ли сохранить результат в виде XPS‑файла?** Да, метод `Save` записывает файл на диск.

## Что такое вертикальный градиент в XPS?

Вертикальный градиент — это переход цвета от верхней части фигуры к нижней. В XPS это достигается с помощью **linear gradient brush**, который определяет начальную и конечную точки, а также набор градиентных остановок, контролирующих цвет в определённых позициях.

## Почему использовать Aspose.Page для создания XPS‑документа с градиентами?

- **Полная интеграция с .NET** – работает с .NET Framework, .NET Core и .NET 5/6+.  
- **Отсутствие внешних зависимостей** – вся отрисовка обрабатывается библиотекой.  
- **Высокая точность** – градиенты отображаются точно так, как определено, в соответствии со спецификацией XPS.  
- **Лёгкое сопровождение** – понятная объектная модель для путей, кистей и цветов.

## Требования

- Библиотека Aspose.Page для .NET: Убедитесь, что библиотека Aspose.Page для .NET установлена в вашей среде разработки. Вы можете скачать её [здесь](https://releases.aspose.com/page/net/).  
- Среда разработки: Настройте .NET‑среду разработки с предпочтительным IDE, например Visual Studio.

Теперь, когда всё готово, перейдём к коду.

## Импорт пространств имён

В вашем .NET‑приложении подключите необходимые пространства имён для доступа к классам и методам Aspose.Page.

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Collections.Generic;
using System.Drawing;
```

## Шаг 1: Настройка каталога документа

Определите папку, в которой будет сохранён сгенерированный XPS‑файл.

```csharp
// ExStart:3
string dataDir = "Your Document Directory";
// ExEnd:3
```

## Шаг 2: Создание нового XPS‑документа

Создайте новый XPS‑документ, который позже будем заполнять путем, заполненным градиентом.

```csharp
// ExStart:4
XpsDocument doc = new XpsDocument();
// ExEnd:4
```

## Шаг 3: Определение градиентных остановок

Градиентные остановки определяют цвета и их позиции вдоль линии градиента. Здесь мы создаём пять остановок для плавного вертикального перехода.

```csharp
// ExStart:5
List<XpsGradientStop> stops = new List<XpsGradientStop>();
stops.Add(doc.CreateGradientStop(doc.CreateColor(253, 255, 12, 0), 0f));
stops.Add(doc.CreateGradientStop(doc.CreateColor(252, 255, 154, 0), 0.359375f));
stops.Add(doc.CreateGradientStop(doc.CreateColor(252, 255, 56, 0), 0.424805f));
stops.Add(doc.CreateGradientStop(doc.CreateColor(253, 255, 229, 0), 0.879883f));
stops.Add(doc.CreateGradientStop(doc.CreateColor(252, 255, 255, 234), 1f));
// ExEnd:5
```

## Шаг 4: Создание пути с градиентом

Мы рисуем путь в виде прямоугольника и применяем **linear gradient brush**, который идёт вертикально от точки (10, 110) до точки (10, 200). Кисть получает градиентные остановки, определённые ранее.

```csharp
// ExStart:6
XpsPath path = doc.AddPath(doc.CreatePathGeometry("M 10,110 L 228,110 228,200 10,200"));
path.RenderTransform = doc.CreateMatrix(1f, 0f, 0f, 1f, 20f, 70f);
path.Fill = doc.CreateLinearGradientBrush(new PointF(10f, 110f), new PointF(10f, 200f));
((XpsGradientBrush)path.Fill).GradientStops.AddRange(stops);
// ExEnd:6
```

## Шаг 5: Сохранение полученного XPS‑документа

Наконец, запишите XPS‑документ на диск. Этот шаг **save XPS file** создаёт `AddVerticalGradient_outXPS.xps` в указанной вами папке.

```csharp
// ExStart:7
doc.Save(dataDir + "AddVerticalGradient_outXPS.xps");
// ExEnd:7
```

**Полезный совет:** Проверьте результат, открыв XPS‑файл в Windows XPS Viewer или любом стороннем просмотрщике, чтобы убедиться, что градиент отображается корректно.

## Распространённые проблемы и их устранение

| Симптом | Возможная причина | Решение |
|---------|-------------------|---------|
| Градиент выглядит как сплошной цвет | Градиентные остановки не добавлены в кисть | Убедитесь, что выполнено `((XpsGradientBrush)path.Fill).GradientStops.AddRange(stops);`. |
| Файл не найден при сохранении | `dataDir` указывает на несуществующую папку | Сначала создайте каталог или используйте абсолютный путь. |
| Цвета выглядят иначе | Значения цвета заданы в порядке ARGB; проверьте порядок каналов | Правильно используйте `CreateColor(alpha, red, green, blue)`. |
