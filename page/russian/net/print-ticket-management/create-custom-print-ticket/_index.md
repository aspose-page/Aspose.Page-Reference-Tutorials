# Как добавить билет: Создание пользовательского печатного билета с Aspose.Page для .NET

## Введение

Если вам нужна функциональность **как добавить билет** в .NET‑приложении, Aspose.Page предоставляет возможность генерировать пользовательские печатные билеты непосредственно из кода. В этом руководстве мы пройдем весь процесс — настройку среды, создание XPS‑документа, прикрепление пользовательского job print ticket и сохранение результата. К концу вы сможете добавить поддержку ticket в любой процесс печати с уверенностью.

## Быстрые ответы
- **Что означает “add ticket”?** Это встраивание пользовательского печатного билета (метаданные XPS), который управляет настройками принтера.  
- **Какая библиотека требуется?** Aspose.Page for .NET.  
- **Нужна ли лицензия?** Временная лицензия подходит для оценки; полная лицензия требуется для продакшн.  
- **Можно ли использовать это с .NET Core?** Да, Aspose.Page поддерживает .NET Framework и .NET Core.  
- **Сколько времени занимает реализация?** Обычно менее 15 минут для базового билета.

## Что такое пользовательский печатный билет?
Пользовательский печатный билет — это описание предпочтений принтера в формате XML (например, сортировка, количество копий, управление цветом и т.д.), которое сопровождает XPS‑документ. Он позволяет разработчикам программно задавать, как документ должен печататься, устраняя необходимость ручной настройки на стороне клиента.

## Почему добавлять поддержку ticket с Aspose.Page?
- **Тонкий контроль:** Устанавливайте сортировку, количество копий, тип носителя и многое другое из кода.  
- **Кроссплатформенная согласованность:** Один и тот же билет работает на любом принтере, поддерживающем метаданные XPS.  
- **Бесшовная интеграция:** Работает напрямую с вашими существующими .NET‑проектами без дополнительных сервисов.

## Требования

Прежде чем погрузиться, убедитесь, что у вас есть:

- Базовые навыки C# и разработки на .NET.  
- Установленная Visual Studio (любая современная версия).  
- Библиотека Aspose.Page for .NET, добавленная в ваш проект.  

Если вы ещё не добавили библиотеку, её можно скачать из [документации Aspose.Page for .NET](https://reference.aspose.com/page/net/). Для помощи сообщества посетите [форум Aspose.Page](https://forum.aspose.com/c/page/39).

## Импорт пространств имён

Начните с подключения пространств имён, которые предоставляют классы XPS и метаданных.

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsMetadata;
using Aspose.Page.XPS.XpsModel;
using System;
using System.Drawing;
```

Теперь разберём реализацию шаг за шагом.

## Шаг 1: Настройка каталога документа

Укажите, где будет сохраняться сгенерированный XPS‑файл.

```csharp
string dir = "Your Document Directory";
```

## Шаг 2: Создание нового XPS‑документа

Создайте новый XPS‑документ, который будет содержать страницы и билет.

```csharp
XpsDocument xDocs = new XpsDocument();
```

## Шаг 3: Добавление пользовательского Job Print Ticket

Прикрепите пользовательский job print ticket к документу. Это ядро функциональности **как добавить билет** — здесь вы указываете сортировку, количество копий, цель рендеринга, управление цветом и любые другие необходимые настройки.

```csharp
xDocs.JobPrintTicket = new JobPrintTicket(
    new PageDevModeSnaphot("SABlAGwAbABvACEAAAA="),
    new DocumentCollate(Collate.CollateOption.Collated),
    // Add other print settings as needed
);
```

> **Pro tip:** Замените строку‑заполнитель snapshot на Base64‑закодированную структуру DEVMODE, соответствующую возможностям вашего принтера.

## Шаг 4: Сохранение документа

Сохраните XPS‑документ (с вложенным билетом) на диск.

```csharp
xDocs.Save(dir + "output1.xps");
```

Когда вы откроете *output1.xps* в просмотрщике, поддерживающем метаданные XPS, принтер автоматически применит настройки, определённые в билете.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| Билет не применяется | Просмотрщик игнорирует метаданные XPS | Используйте драйвер принтера, поддерживающий XPS‑print tickets, или просмотрщик, например Microsoft XPS Viewer. |
| Недействительный Base64‑snapshot | Повреждённые данные DEVMODE | Воссоздайте snapshot из драйвера принтера с помощью API `GetPrinter`. |
| Отсутствуют права | Запись в `dir` запрещена | Убедитесь, что приложение запускается с достаточными правами доступа к файловой системе. |
