## Qu'est-ce qu'un document XPS et pourquoi en créer un ?

XPS (XML Paper Specification) est un format de document à mise en page fixe développé par Microsoft. Il préserve la mise en page, les polices et les graphiques sur toutes les plateformes, ce qui le rend idéal pour les factures, les rapports et les formulaires imprimables. Créer des documents XPS de manière programmatique vous donne un contrôle total sur la conception et l'automatisation, éliminant ainsi la rédaction manuelle.

- **Haute fidélité** – les fichiers XPS générés s'affichent de façon identique sous Windows, Linux et les plateformes mobiles.  
- **Support multi‑format** – la même base de code peut également générer des fichiers PostScript, permettant des flux de travail « écrire une fois, produire plusieurs sorties ».

## Questions fréquentes

**Q : Comment démarrer un nouveau document XPS à partir de zéro ?**  
R : Instanciez la classe `Document`, ajoutez une `Page`, puis utilisez les objets `Graphics` pour dessiner du texte, des images ou des formes.

**Q : Puis-je convertir un PDF existant en XPS avec Aspose.Page ?**  
R : La conversion directe PDF‑vers‑XPS est gérée par Aspose.PDF, mais vous pouvez exporter le contenu du PDF en images et les intégrer dans un document XPS avec Aspose.Page.

**Q : Est-il possible de modifier un fichier XPS existant sans le recréer ?**  
R : Oui – chargez le fichier avec `Document.Load`, modifiez les pages ou ajoutez du nouveau contenu, puis enregistrez-le.

**Q : Quelle est la meilleure façon de générer un fichier PostScript pour l'impression ?**  
R : Utilisez la même API `Document`, mais appelez `Save` avec l'option `SaveFormat.PostScript`. Cela garantit une sortie prête à imprimer.

**Q : Existe-t-il des limites de taille pour les fichiers XPS ou PostScript ?**  
R : La bibliothèque gère efficacement les gros fichiers ; toutefois, pour des documents extrêmement volumineux, envisagez de diffuser le contenu et d'utiliser `Document.OptimizeResources()`.
