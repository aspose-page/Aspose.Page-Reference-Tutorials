## Common Pitfalls & Tips

- **La taille de l'image compte** – Les grandes images augmentent la consommation de mémoire. Redimensionnez ou compressez avant d'ajouter.
- **Système de coordonnées** – XPS utilise 96 DPI ; pensez à convertir les valeurs en pixels en points lors du positionnement.
- **Transparence** – Les images PNG conservent les canaux alpha, mais assurez-vous que le visualiseur cible prend en charge la transparence.
- **Débogage du placement** – Utilisez `saveAsPdf()` pour générer un aperçu PDF rapide et vérifier les coordonnées.

## Frequently Asked Questions

**Q : Puis-je ajouter des images à un fichier XPS existant ?**  
R : Oui. Ouvrez le XPS existant avec `XpsDocument` et utilisez les mêmes API `addImage` pour insérer de nouveaux graphiques.

**Q : Quelles limites de taille d'image s'appliquent ?**  
R : Aspose.Page prend en charge les images jusqu'à 10 Mo ; les fichiers plus volumineux peuvent affecter les performances mais restent traitables.

**Q : Le carrelage affecte-t-il la taille du fichier ?**  
R : Le carrelage répète les mêmes données d'image, donc la taille du fichier augmente très peu — seule la définition de la mosaïque est stockée.

**Q : Existe-t-il un moyen de conserver la transparence de l'image ?**  
R : Les images PNG conservent les canaux alpha lorsqu'elles sont ajoutées, ainsi les zones transparentes sont rendues correctement dans le XPS.

**Q : Comment déboguer les problèmes de placement d'image ?**  
R : Utilisez la méthode `saveAsPdf` pour exporter un PDF d'aperçu ; cela aide à visualiser les coordonnées et l'échelle avant la sortie finale XPS.

**Q : Que faire si je dois faire pivoter ou redimensionner une image ?**  
R : Appliquez une matrice de transformation via l'objet `Graphics` avant d'appeler `addImage`.

**Q : Puis-je combiner plusieurs images sur une même page ?**  
R : Absolument. Appelez `addImage` à plusieurs reprises avec différents rectangles ou paramètres de transformation.
