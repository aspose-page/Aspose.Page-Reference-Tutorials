# كيفية قص XPS باستخدام Aspose.Page لـ .NET

## المقدمة

مرحبًا بكم في هذا الدرس الشامل حول **كيفية قص XPS** باستخدام Aspose.Page لـ .NET! في هذا الدليل، سنرشدكم خلال عملية إنشاء مستندات XPS، تعديلها، وحفظها باستخدام المكتبة. XPS، أو مواصفة الورق XML، هو تنسيق مستند موحد ومفتوح، وتوفر Aspose.Page لـ .NET أدوات قوية للعمل مع مستندات XPS في تطبيقات .NET الخاصة بكم.

## إجابات سريعة
- **ما هو قص XPS؟** تطبيق قناع هندسي (قص) لتحديد المنطقة المرئية لعناصر قماش XPS.  
- **أي مكتبة هي الأفضل لهذا؟** Aspose.Page لـ .NET تقدم واجهة برمجة تطبيقات كاملة لإنشاء وقص XPS.  
- **المتطلبات المسبقة؟** Visual Studio، بيئة تشغيل .NET، ومكتبة Aspose.Page لـ .NET.  
- **كم يستغرق التنفيذ؟** تقريبًا 10‑15 دقيقة لسيناريو قص أساسي.  
- **هل يمكن استخدامه في الإنتاج؟** نعم، مع ترخيص Aspose صالح (يتوفر نسخة تجريبية).

## ما هو “كيفية قص XPS”؟
يعمل القص في XPS عن طريق تعريف **هندسة قص** (مثلاً دائرة أو مستطيل) وتعيينها إلى القماش. أي شيء يُرسم خارج تلك الهندسة لا يتم عرضه، مما يتيح لك إنشاء تخطيطات صفحات متقدمة مثل الصور المقنّعة، الأشكال المخصصة، أو مناطق المحتوى المركّزة.

## لماذا نستخدم Aspose.Page لـ .NET لقص XPS؟
- **تحكم كامل** في تحويلات القماش، هندسات المسار، والفرش.  
- **بدون تبعيات خارجية** – كل شيء يعمل داخل تطبيق .NET الخاص بك.  
- **دعم متعدد المنصات** لـ .NET Framework، .NET Core، و .NET 5/6+.  
- **أداء عالي** بواجهة برمجة تطبيقات خفيفة الوزن تكتب ملفات XPS صالحة.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من توفر ما يلي:

- تثبيت Visual Studio على جهازك.  
- إضافة مكتبة Aspose.Page لـ .NET إلى مشروعك. يمكنك تنزيلها [هنا](https://releases.aspose.com/page/net/).  
- معرفة أساسية بلغة البرمجة C#.

## استيراد مساحات الأسماء

لاستخدام وظائف Aspose.Page لـ .NET، تحتاج إلى استيراد مساحات الأسماء المطلوبة إلى مشروعك. اتبع الخطوات التالية:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

الآن، لنقسم مثال الشيفرة الذي قدمته إلى عدة خطوات.

## الخطوة 1: تعيين مسار دليل المستندات.

```csharp
string dataDir = "Your Document Directory";
```

تأكد من استبدال "Your Document Directory" بالمسار الفعلي لدليل المستندات الخاص بك.

## الخطوة 2: إنشاء مستند XPS جديد.

```csharp
XpsDocument doc = new XpsDocument();
```

هذا ينشئ مستند XPS جديد ستعمل عليه.

## الخطوة 3: إنشاء القماش الرئيسي.

```csharp
XpsCanvas canvas1 = doc.AddCanvas();
```

هذه الخطوة تنشئ القماش الرئيسي، وهو مشترك لجميع عناصر الصفحة.

## الخطوة 4: تعيين إزاحات اليسار والعلو في القماش الرئيسي.

```csharp
canvas1.RenderTransform = doc.CreateMatrix(1, 0, 0, 1, 20, 10);
```

قم بضبط إزاحات اليسار والعلو وفقًا لمتطلباتك.

## الخطوة 5: إنشاء هندسة مسار مستطيل.

```csharp
XpsPathGeometry rectGeom = doc.CreatePathGeometry("M 0,0 L 500,0 500,300 0,300 Z");
```

هذا ينشئ هندسة مسار لمستطيل.

## الخطوة 6: إنشاء تعبئة للمستطيلات.

```csharp
XpsBrush fill = doc.CreateSolidColorBrush(doc.CreateColor(12, 15, 159));
```

حدد لون التعبئة للمستطيلات.

## الخطوة 7: إضافة قماش آخر مع قص إلى القماش الرئيسي.

```csharp
XpsCanvas canvas2 = canvas1.AddCanvas();
```

هذه الخطوة تضيف قماشًا آخر إلى القماش الرئيسي.

## الخطوة 8: إنشاء هندسة دائرة للقص.

```csharp
XpsPathGeometry clipGeom = doc.CreatePathGeometry("M250,250 A100,100 0 1 1 250,50 100,100 0 1 1 250,250");
canvas2.Clip = clipGeom;
```

هذا ينشئ هندسة قص دائرية ويطبقها على القماش الثاني.

## الخطوة 9: إنشاء مستطيل في القماش الثاني وتعبئته.

```csharp
XpsPath rect = canvas2.AddPath(rectGeom);
rect.Fill = fill;
```

هذه الخطوة تنشئ مستطيلًا في القماش الثاني وتملأه.

## الخطوة 10: إضافة القماش الثاني مع مستطيل مُحدَّد إلى القماش الرئيسي.

```csharp
XpsCanvas canvas3 = canvas1.AddCanvas();
```

هذا يضيف قماشًا آخر إلى القماش الرئيسي.

## الخطوة 11: إنشاء مستطيل في القماش الثالث وتحديد حدوده.

```csharp
rect = canvas3.AddPath(rectGeom);
rect.Stroke = fill;
rect.StrokeThickness = 2;
```

هذا ينشئ مستطيلًا في القماش الثالث ويطبق حدًا عليه.

## الخطوة 12: حفظ مستند XPS الناتج.

```csharp
doc.Save(dataDir + "output2.xps");
```

هذا يحفظ مستند XPS إلى الدليل المحدد.

## المشكلات الشائعة والحلول
- **مسار غير صالح** – تأكد من أن `dataDir` ينتهي بشرطة مائلة عكسية (`\\`) أو استخدم `Path.Combine`.  
- **القص غير مطبق** – تحقق من أن سلسلة هندسة القص مُشكَّلة بشكل صحيح؛ فقد يؤدي نقص مسافة إلى تجاهل القص.  
- **استثناء الترخيص** – في بناء غير تجريبي، أضف ترخيص Aspose صالح قبل إنشاء المستند لتجنب استثناءات وقت التشغيل.

## أسئلة متكررة إضافية

**س: هل يمكنني دمج عدة هندسات قص على قماش واحد؟**  
ج: نعم، يمكنك تعيين `PathGeometry` معقد يحتوي على عدة مسارات فرعية إلى خاصية `Clip`، مما يسمح بإنشاء أقنعة متعددة الطبقات.

**س: هل يؤثر القص على تحويل PDF؟**  
ج: عند تحويل XPS إلى PDF باستخدام Aspose.PDF، يتم الحفاظ على هندسة القص، لذا يبقى الناتج البصري متطابقًا.

**س: هل يمكن تحريك القص في XPS؟**  
ج: لا يدعم XPS الرسوم المتحركة؛ ومع ذلك، يمكنك إنشاء سلسلة من صفحات XPS بأشكال قص مختلفة لمحاكاة الحركة.
