## ما هو XMP metadata؟
XMP (Extensible Metadata Platform) هو تنسيق موحد قائم على XML لتضمين بيانات وصفية غنية—مثل المُنشئ، العنوان، والخصائص المخصصة—مباشرة داخل الملفات. في مستندات EPS، يعيش XMP جنبًا إلى جنب مع تعليقات PostScript التقليدية، مما يتيح للتطبيقات قراءة وتعديل البيانات الوصفية دون تغيير المحتوى البصري.

## لماذا تستخدم Aspose.Page for Java لتعديل XMP؟

## استيراد الحزم
تعليمات `import` تجلب فئات Aspose.Page الأساسية إلى النطاق.

فئة `PsDocument` هي الكائن الأساسي في Aspose.Page الذي يمثل مستند EPS في الذاكرة.  
فئة `XmpMetadata` توفر الوصول إلى حزمة XMP المدمجة في ملف EPS.  

```java
import java.io.FileInputStream;
import java.io.FileOutputStream;
import com.aspose.eps.PsDocument;
import com.aspose.eps.xmp.XmpMetadata;
import com.aspose.eps.xmp.XmpValue;
import com.aspose.page.BaseExamplesTest;
import com.aspose.page.License;
```

## كيفية تعديل بيانات XMP الوصفية؟
حمّل ملف EPS، استرجع حزمة XMP الخاصة به، عدّل الخاصية المطلوبة، واحفظ المستند—كل ذلك في بضع أسطر بسيطة. هذا النهج يعمل مع أي قيمة XMP مسماة، سواء كانت حقلًا قياسيًا من Dublin Core أو مساحة أسماء مخصصة قمت بتعريفها.

## الخطوة 1: تهيئة تدفق ملف EPS الإدخالي
`FileInputStream` يفتح تدفقًا للقراءة فقط إلى ملف EPS المصدر، مما يسمح لـ Aspose.Page بقراءة المستند دون قفل نظام الملفات.

```java
// The path to the documents directory.
String dataDir = "Your Document Directory";
        
// Initialize input EPS file stream
FileInputStream psStream = new FileInputStream(dataDir + "xmp4.eps");
```

## الخطوة 2: تهيئة PsDocument
`PsDocument` هو الكائن الأساسي الذي يضم محتوى EPS ويوفر الوصول إلى بياناته الوصفية والصفحات والموارد.

```java
PsDocument document = new PsDocument(psStream);
```

## الخطوة 3: الحصول على XMP Metadata
`XmpMetadata` يمثل حزمة XMP داخل ملف EPS ويقدم طرقًا لقراءة أو تعديل أو حذف الخصائص الفردية.

```java
// Get XMP metadata. If EPS file doesn't contain XMP metadata, we get a new one filled with values from PS metadata comments (%%Creator, %%CreateDate, %%Title, etc.)
XmpMetadata xmp = document.getXmpMetadata();
```

## الخطوة 4: تعيين قيمة جديدة في XMP
`setProperty` يحدد قيمة خاصية XMP معينة في حزمة البيانات الوصفية.

```java
// Set new string value "Inches" for named value "stDim:unit" of structure "xmpTPg:MaxPageSize" 
xmp.setNamedValue("xmpTPg:MaxPageSize", "stDim:unit", new XmpValue("Inches"));
```

## الخطوة 5: تهيئة تدفق ملف EPS الإخراجي
`FileOutputStream` ينشئ تدفقًا قابلًا للكتابة لحفظ ملف EPS المعدل.

```java
// Initialize output EPS file stream
FileOutputStream outPsStream = new FileOutputStream(dataDir + "xmp4_changed.eps");
```

## الخطوة 6: حفظ المستند مع بيانات XMP الوصفية المعدلة
`save` يكتب كائن PsDocument الموجود في الذاكرة، بما في ذلك XMP المحدث، إلى تدفق الإخراج.

```java
// Save document with changed XMP metadata
try {			
    document.save(outPsStream);
} finally {
    outPsStream.close();
}
```

## الخطوة 7: إغلاق تدفق EPS الإدخالي
`close` يحرر مقبض الملف ويحرر الموارد المرتبطة بتدفق الإدخال.

```java
// Close input EPS stream
psStream.close();
```

## المشكلات الشائعة والحلول
- **FileNotFoundException** – تحقق من أن `dataDir` يشير إلى المجلد الصحيح وأن `xmp4.eps` موجود.  
- **LicenseException** – إذا ظهرت أخطاء ترخيص، تأكد من تحميل ملف ترخيص Aspose.Page صالح قبل إنشاء `PsDocument`.  
- **Metadata Not Updated** – بعد الحفظ، افتح الـ EPS الناتج في عارض بيانات وصفية (مثل Adobe Bridge) لتأكيد ظهور القيمة الجديدة.
