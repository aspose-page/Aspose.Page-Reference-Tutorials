## Java で Aspose.Page を使用して EPS をリサイズする方法
EPS ファイルのリサイズは、事前に定義されたレイアウトにグラフィックを合わせたり、転送速度向上のためにファイルサイズを削減したりする際の最初のステップになることが多いです。以下にプロセスの簡潔な手順を示します。実際のコードスニペットは、このページ後半の専用リサイズチュートリアルで提供されています。

1. **Load the EPS document** – `Page` クラスを使用してソースファイルを開きます。  
2. **Define the new dimensions** – 目的の幅と高さをポイント単位で設定します（1 pt = 1/72 インチ）。  
3. **Apply the resize operation** – `page.resize()` を呼び出すか、`PageInfo` を介してページサイズを調整します。  
4. **Save the result** – EPS、PDF、PNG、または他のサポートされている形式にエクスポートします。

> **Pro tip:** 印刷用にサイズだけ変更する場合は、元の EPS をそのまま残し、一時的なリサイズコピーを生成してください。これにより、マスターファイルを将来再利用できる状態で保持できます。

## 一般的な問題とヒント
- **Missing fonts** – EPS が参照するフォントがホストマシンにインストールされているか、API を使用して埋め込んでいることを確認してください。  
- **Large file size after resize** – `saveOptions.setCompressionLevel()` メソッドを使用して、品質を損なうことなくファイルサイズを削減します。  
- **Unexpected clipping** – クロップ矩形の座標を再確認してください。座標はポイント単位で表されます（1 pt = 1/72 インチ）。

## よくある質問
**Q: 複数の EPS ファイルをバッチ処理できますか？**  
A: はい。クロッピングまたはリサイズのロジックをループで囲み、各ファイルに同じ `Page` オブジェクトを再利用してください。

**Q: Aspose.Page は埋め込みラスタ画像を含む EPS をサポートしていますか？**  
A: もちろんです。ベクター要素とラスタ要素の両方が操作中に保持されます。

**Q: 編集後に EPS を他の形式（PDF、PNG）に変換できますか？**  
A: はい。クロッピングまたはリサイズ後に、`page.save("output.pdf", SaveFormat.Pdf)` や `SaveFormat.Png` を呼び出すことで変換できます。

**Q: EPS ファイルが破損している場合はどうなりますか？**  
A: API は `FileFormatException` をスローします。例外を捕捉し、処理前にソースファイルを検証してください。

**Q: リソースを手動で閉じる必要がありますか？**  
A: `Page` クラスは `Closeable` を実装しています。try‑with‑resources ブロックを使用するか、`page.close()` を呼び出してメモリを解放してください。
