# Aspose.Page テキスト操作

## はじめに

ようこそ、.NET 用 **aspose.page text manipulation** の世界へ。この包括的なガイドでは、PostScript (PS) および XPS ドキュメント内にテキストを追加、編集、スタイル設定するために必要な基本テクニックをご紹介します。レポーティングエンジンの構築、印刷可能な請求書の生成、カスタムグラフィックの作成など、これらの機能を習得すれば .NET プロジェクトがより強力で柔軟になります。

## クイック回答
- **What can I manipulate?** PostScript と XPS の両方のドキュメントにプレーンテキストまたは Unicode テキストを追加できます。  
- **Which library is required?** Aspose.Page for .NET がテキスト操作 API のフルセットを提供します。  
- **Do I need a license?** 開発用途は無料トライアルで可能です。商用利用には製品ライセンスが必要です。  
- **Supported platforms?** .NET Framework 4.5 以上、.NET Core 3.1 以上、.NET 5/6/7。  
- **How long does it take?** 簡単なテキスト挿入は 10 分未満で実装できます。

## Aspose.Page テキスト操作とは？

Aspose.Page のテキスト操作機能を使うと、文字列の配置、フォント制御、Unicode 文字の適用、ページ上への正確な位置指定をプログラムで行えます。API は低レベルな PS/XPS 構文を抽象化するため、ドキュメント内部の詳細に煩わされることなくビジネスロジックに集中できます。

## なぜ Aspose.Page をテキスト操作に使用するのか？

- **Cross‑format support** – 一度書くだけで PostScript と XPS の両方に出力できます。  
- **Full Unicode handling** – 国際文書で文字欠損が起きません。  
- **No external dependencies** – ネイティブバイナリ不要の純粋 .NET ライブラリです。  
- **High performance** – 大規模なドキュメント生成に最適化されています。

## 前提条件
- Visual Studio 2022 または任意の .NET 対応 IDE。  
- .NET 6 SDK（または .NET Framework 4.5 以上）。  
- Aspose.Page for .NET NuGet パッケージをインストール (`Install-Package Aspose.Page`)。

## よくある質問

**Q:** *Can I use these tutorials in a commercial application?*  
**A:** はい。商用利用には有効な Aspose.Page ライセンスが必要です。評価目的であれば無料トライアルをご利用いただけます。

**Q:** *Does Aspose.Page support right‑to‑left languages?*  
**A:** もちろんです。Unicode の取り扱いにはアラビア語やヘブライ語などの RTL スクリプトも含まれます。

**Q:** *What if I need to add text to an existing PS/XPS file?*  
**A:** API を使用すれば既存のドキュメントを読み込み、ページを変更し、全体を再作成せずに結果を保存できます。

**Q:** *Is there a way to measure text dimensions before placing it?*  
**A:** はい。`TextFragment` の測定メソッドを使って、選択したフォントに基づく幅と高さを計算できます。

**Q:** *Do I need to manage fonts manually?*  
**A:** Aspose.Page はシステムフォントを自動で埋め込むことができますが、プラットフォーム間での一貫したレンダリングを実現するためにカスタムフォントファイルを指定することも可能です。
