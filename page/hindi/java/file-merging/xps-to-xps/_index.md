# Java में Aspose.Page के साथ XPS फ़ाइलें कैसे मर्ज करें

XPS दस्तावेज़ों को मर्ज करना एक सामान्य कार्य है जब आपको रिपोर्ट, प्रेज़ेंटेशन या किसी भी XPS फ़ाइलों के संग्रह को एक ही, आसानी से‑शेयर करने योग्य पैकेज में संयोजित करना हो। इस ट्यूटोरियल में आप **XPS फ़ाइलें कैसे मर्ज करें** सीखेंगे Aspose.Page for Java API का उपयोग करके, स्पष्ट व्याख्याओं, वास्तविक‑विश्व टिप्स और तैयार‑कोड स्निपेट्स के साथ।

## त्वरित उत्तर
- **XPS मर्जिंग को कौनसी लाइब्रेरी संभालती है?** Aspose.Page for Java.  
- **इम्प्लीमेंटेशन में कितना समय लगेगा?** बेसिक मर्ज के लिए लगभग 10‑15 मिनट।  
- **टेस्टिंग के लिए लाइसेंस चाहिए?** हाँ – Aspose से एक अस्थायी ट्रायल लाइसेंस उपलब्ध है।  
- **क्या अलग‑अलग पेज काउंट वाली फ़ाइलें मर्ज कर सकते हैं?** बिल्कुल; Aspose.Page किसी भी वैध XPS दस्तावेज़ को मर्ज करता है।  
- **कौनसे Java संस्करण समर्थित हैं?** Java 8 और उससे ऊपर (JDK 11+ की सलाह दी जाती है)।

## XPS फ़ाइल मर्जिंग क्या है?
XPS (XML Paper Specification) माइक्रोसॉफ्ट का फिक्स्ड‑लेआउट दस्तावेज़ फ़ॉर्मेट है। XPS फ़ाइलें मर्ज करने का मतलब कई XPS दस्तावेज़ों को एक ही फ़ाइल में जोड़ना है, जबकि प्रत्येक पेज का लेआउट, फ़ॉन्ट और ग्राफ़िक्स बरकरार रहता है।

## Java में XPS फ़ाइलें क्यों मर्ज करें?
- **ऑटोमेशन:** कई मॉड्यूल से एक ही रिपोर्ट जनरेट करें बिना मैन्युअल हस्तक्षेप के।  
- **संगति:** मूल XPS पेजों की सटीक विज़ुअल फ़िडेलिटी बनाए रखें।  
- **प्रदर्शन:** ट्रांसफ़र या स्टोरेज के लिए फ़ाइलों की संख्या कम करें।  
- **क्रॉस‑प्लेटफ़ॉर्म:** Java आपको Windows, Linux या macOS सर्वरों पर मर्ज चलाने की सुविधा देता है।

- **Java Development Kit (JDK):** सुनिश्चित करें कि आपके सिस्टम पर JDK स्थापित है। आप इसे [यहाँ](https://www.oracle.com/java/technologies/javase-downloads.html) से डाउनलोड कर सकते हैं।  
- **Aspose.Page for Java:** Aspose.Page for Java लाइब्रेरी को [Aspose वेबसाइट](https://purchase.aspose.com/buy) से डाउनलोड और इंस्टॉल करें।  
- **Integrated Development Environment (IDE):** अपनी पसंदीदा IDE चुनें; लोकप्रिय विकल्पों में Eclipse, IntelliJ IDEA या NetBeans शामिल हैं।

अब जब सब सेट है, चलिए कोड में डुबकी लगाते हैं।

## पैकेज इम्पोर्ट करें
अपने Java प्रोजेक्ट में, Aspose.Page for Java का उपयोग करने के लिए आवश्यक पैकेज इम्पोर्ट करें। अपने Java फ़ाइल की शुरुआत में निम्नलाइनों को जोड़ें:

```java
import java.io.FileOutputStream;

import com.aspose.xps.XpsDocument;
```

## चरण 1: अपना प्रोजेक्ट सेट अप करें
अपने चुने हुए IDE में एक नया Java प्रोजेक्ट बनाएं और Aspose.Page JAR फ़ाइलों को प्रोजेक्ट के बिल्ड पाथ में जोड़ें। इससे कंपाइलर `XpsDocument` क्लास को ढूँढ सकेगा।

## चरण 2: XPS आउटपुट स्ट्रीम को इनिशियलाइज़ करें
मर्ज्ड XPS फ़ाइल के लिए आउटपुट स्ट्रीम सेट करें। वह डायरेक्टरी निर्दिष्ट करें जहाँ आप मर्ज्ड फ़ाइल को सेव करना चाहते हैं।

```java
String dataDir = "Your Document Directory";
FileOutputStream outStream = new FileOutputStream(dataDir + "mergedXPSfiles.xps");
```

> **Pro tip:** विकास के दौरान `FileNotFoundException` से बचने के लिए एक एब्सोल्यूट पाथ उपयोग करें, फिर प्रोडक्शन में रिलेटिव पाथ पर स्विच करें।

## चरण 3: पहली XPS फ़ाइल लोड करें
पहली XPS फ़ाइल लोड करें जो मर्जिंग के बेस के रूप में काम करेगी।

```java
XpsDocument document = new XpsDocument(dataDir + "input.xps");
```

पहले दस्तावेज़ की प्रॉपर्टीज़ (जैसे पेज साइज और ओरिएंटेशन) अंतिम मर्ज्ड फ़ाइल के डिफ़ॉल्ट बन जाती हैं।

## चरण 4: XPS फ़ाइलों की एरे बनाएं
उन XPS फ़ाइलों की एरे तैयार करें जिन्हें आप पहली फ़ाइल के साथ मर्ज करना चाहते हैं।

```java
String[] filesForMerge = new String[] { dataDir + "Demo.xps", dataDir + "sample.xps" };
```

आप जितने चाहें फ़ाइल पाथ जोड़ सकते हैं; एरे को डायरेक्टरी लिस्टिंग से डायनामिक रूप से भी बनाया जा सकता है।

## चरण 5: मर्ज करें और सेव करें
मर्जिंग प्रक्रिया को निष्पादित करें और परिणाम को निर्दिष्ट आउटपुट स्ट्रीम में सेव करें।

```java
document.merge(filesForMerge, outStream);
```

इस कॉल के बाद, `mergedXPSfiles.xps` में `input.xps`, `Demo.xps` और `sample.xps` की सभी पेज़ें आपके द्वारा निर्दिष्ट क्रम में होंगी।

## सामान्य समस्याएँ और समाधान
| समस्या | कारण | समाधान |
|-------|--------|-----|
| **`FileNotFoundException`** | गलत `dataDir` पाथ | फ़ोल्डर मौजूद है या नहीं जांचें और Windows पर डबल बैकस्लैश (`\\`) उपयोग करें। |
| **License not found** | वैध लाइसेंस के बिना चलाना | Aspose से अस्थायी लाइसेंस लागू करें या पूर्ण लाइसेंस खरीदें। |
| **Merged file is empty** | आउटपुट स्ट्रीम फ़्लश/क्लोज नहीं हुआ | `document.merge(...)` के बाद `outStream.close()` कॉल करें। |
| **Mismatched page sizes** | स्रोत XPS फ़ाइलों के आयाम अलग हैं | मर्ज करने से पहले `document.setPageSize(...)` उपयोग करके एक समान साइज सेट करें। |
