# Aspose.Page for .NET के साथ XPS दस्तावेज़ों को कैसे मर्ज करें

## परिचय

क्या आप प्रोग्रामेटिक रूप से **XPS फ़ाइलों को मर्ज करने** का विश्वसनीय तरीका खोज रहे हैं? इस ट्यूटोरियल में हम आपको Aspose.Page for .NET का उपयोग करके XPS दस्तावेज़ों को मर्ज करने के सटीक चरणों से परिचित कराएंगे। चाहे आपको रिपोर्ट, इनवॉइस, या किसी भी अन्य XPS‑आधारित एसेट को संयोजित करना हो, प्रक्रिया सरल और पूरी तरह स्वचालित है। चलिए देखते हैं कि आप कुछ ही C# कोड लाइनों के साथ साफ़, मर्ज किया हुआ XPS आउटपुट कैसे प्राप्त कर सकते हैं।

## त्वरित उत्तर
- **XPS मर्जिंग को कौन सी लाइब्रेरी संभालती है?** Aspose.Page for .NET  
- **इम्प्लीमेंटेशन में कितना समय लगता है?** आमतौर पर 10 मिनट से कम  
- **क्या मुझे लाइसेंस चाहिए?** प्रोडक्शन उपयोग के लिए लाइसेंस आवश्यक है; एक मुफ्त ट्रायल उपलब्ध है  
- **समर्थित .NET संस्करण?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7  
- **क्या मैं एन्क्रिप्टेड XPS फ़ाइलों को मर्ज कर सकता हूँ?** हाँ – Aspose.Page एन्क्रिप्टेड दस्तावेज़ों को संभाल सकता है  

## XPS दस्तावेज़ मर्जिंग क्या है?
XPS (XML Paper Specification) माइक्रोसॉफ्ट द्वारा निर्मित एक फिक्स्ड‑लेआउट दस्तावेज़ फ़ॉर्मेट है। XPS फ़ाइलों को मर्ज करना का अर्थ है कई XPS दस्तावेज़ों को एक ही निरंतर फ़ाइल में जोड़ना, जबकि मूल लेआउट, फ़ॉन्ट और ग्राफ़िक्स को संरक्षित रखना।

## .NET के लिए Aspose.Page क्यों उपयोग करें?
- **पूर्ण नियंत्रण** मर्ज प्रक्रिया पर, बिना Microsoft XPS Viewer की आवश्यकता के  
- **कोई बाहरी निर्भरताएँ नहीं** – सब कुछ आपके .NET एप्लिकेशन के भीतर चलता है  
- **उच्च प्रदर्शन** – बड़े दस्तावेज़ों के साथ भी कुशलता से काम करता है  
- **क्रॉस‑प्लेटफ़ॉर्म** – .NET Framework, .NET Core, और .NET 5+ के साथ संगत  

## पूर्वापेक्षाएँ
शुरू करने से पहले सुनिश्चित करें कि आपके पास है:

- C# और .NET इकोसिस्टम की बुनियादी समझ।  
- **Aspose.Page for .NET** स्थापित है – आप इसे [यहाँ](https://releases.aspose.com/page/net/) से डाउनलोड कर सकते हैं।  
- एक या अधिक XPS फ़ाइलें जिन्हें आप संयोजित करना चाहते हैं।

## नेमस्पेस इम्पोर्ट करें
अपने C# प्रोजेक्ट में, वह नेमस्पेस इम्पोर्ट करें जो आपको XPS API तक पहुंच देता है:

```csharp
using Aspose.Page.XPS;
```

## चरण 1: अपने प्रोजेक्ट को सेट अप करें
Visual Studio, Rider, या अपने पसंदीदा IDE में एक नया C# कंसोल या लाइब्रेरी प्रोजेक्ट बनाएं। Aspose.Page DLL (या NuGet पैकेज `Aspose.Page`) का रेफ़रेंस जोड़ें। इससे आपको बाद में उपयोग होने वाली `XpsDocument` क्लास तक पहुंच मिलेगी।

## चरण 2: स्ट्रीम्स को इनिशियलाइज़ करें
स्रोत XPS फ़ाइलों को इनपुट स्ट्रीम के रूप में खोलें और मर्ज किए गए दस्तावेज़ के लिए एक आउटपुट स्ट्रीम बनाएं। `using` स्टेटमेंट यह सुनिश्चित करते हैं कि ऑपरेशन के बाद सभी स्ट्रीम सही ढंग से बंद हो जाएँ।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize XPS output stream
using (System.IO.Stream outStream = System.IO.File.Open(dataDir + "mergedXPSfiles.xps", System.IO.FileMode.OpenOrCreate, System.IO.FileAccess.Write))
// Initialize XPS input stream
using (System.IO.Stream inStream = System.IO.File.Open(dataDir + "input.xps", System.IO.FileMode.Open))
```

## चरण 3: XPS दस्तावेज़ लोड करें
प्राथमिक इनपुट स्ट्रीम से एक `XpsDocument` इंस्टेंस बनाएं। यदि आवश्यक हो तो `XpsLoadOptions` ऑब्जेक्ट के माध्यम से लोडिंग व्यवहार को कस्टमाइज़ कर सकते हैं।

```csharp
XpsDocument document = new XpsDocument(inStream, new XpsLoadOptions());
```

## चरण 4: XPS फ़ाइलों की एरे बनाएं
एक स्ट्रिंग एरे तैयार करें जिसमें आप मर्ज करना चाहते प्रत्येक XPS फ़ाइल का पथ हो। एरे का क्रम अंतिम दस्तावेज़ में फ़ाइलों के क्रम को निर्धारित करता है।

```csharp
string[] filesToMerge = new string[] { dataDir + "Demo.xps", dataDir + "sample.xps" };
```

## चरण 5: XPS फ़ाइलों को मर्ज करें
`Merge` मेथड को कॉल करें, जिसमें फ़ाइल पाथ की एरे और आउटपुट स्ट्रीम पास करें। Aspose.Page सभी जटिल कार्य संभालता है—पेजों को जोड़ना, रिसोर्सेज़ को संरक्षित रखना, और अंतिम XPS फ़ाइल लिखना।

```csharp
document.Merge(filesToMerge, outStream);
```

## सामान्य समस्याएँ और टिप्स
- **File not found** – `filesToMerge` में पाथ दोबारा जांचें। `Path.Combine` का उपयोग करने से पाथ‑सेपरेटर की गलतियों से बचा जा सकता है।  
- **Memory usage** – बड़ी संख्या में फ़ाइलों को मर्ज करते समय, मेमोरी उपयोग कम रखने के लिए उन्हें बैच में प्रोसेस करने पर विचार करें।  
- **Encrypted documents** – यदि कोई स्रोत XPS पासवर्ड‑प्रोटेक्टेड है, तो मर्ज करने से पहले उपयुक्त क्रेडेंशियल्स के साथ उसे लोड करें।  

## अक्सर पूछे जाने वाले प्रश्न
**Q1:** क्या मैं विभिन्न पेज आकारों वाली XPS फ़ाइलों को मर्ज कर सकता हूँ?  
A: हाँ। Aspose.Page मर्ज के दौरान पेज आयामों को स्वचालित रूप से सामान्य करता है।

**Q2:** क्या XPS फ़ाइलों की संख्या पर कोई सीमा है?  
A: कोई कठोर सीमा नहीं है, लेकिन बहुत बड़ी कलेक्शन प्रदर्शन को प्रभावित कर सकती है; मेमोरी उपयोग पर नज़र रखें।

**Q3:** एन्क्रिप्टेड XPS दस्तावेज़ों को मर्ज करने के लिए क्या विशेष लाइसेंस चाहिए?  
A: उत्पादन‑स्तर की किसी भी सुविधा के लिए, जिसमें एन्क्रिप्टेड दस्तावेज़ संभालना शामिल है, पूर्ण Aspose.Page लाइसेंस आवश्यक है।

**Q4:** मर्ज के बाद प्रत्येक पेज में कस्टम फुटर कैसे जोड़ें?  
A: मर्ज के बाद, आप परिणामी XPS को `XpsDocument` से पुनः खोल सकते हैं और ड्राइंग API का उपयोग करके फुटर डाल सकते हैं।

**Q5:** क्या Aspose.Page .NET Core को सपोर्ट करता है?  
A: बिल्कुल। लाइब्रेरी .NET Core 3.1 और बाद के संस्करणों, साथ ही .NET 5/6/7 के साथ संगत है।

## निष्कर्ष
अब आपने Aspose.Page for .NET का उपयोग करके **XPS दस्तावेज़ों को मर्ज करने** का प्रभावी तरीका सीख लिया है। ऊपर दिए गए चरणों का पालन करके आप किसी भी .NET एप्लिकेशन में दस्तावेज़ समेकन को स्वचालित कर सकते हैं, जिससे समय बचता है और मैन्युअल प्रयास कम होता है। API को और अधिक एक्सप्लोर करें ताकि आप वॉटरमार्क जोड़ सकें, अंतिम फ़ाइल को एन्क्रिप्ट कर सकें, या आवश्यकतानुसार व्यक्तिगत पेजों को संशोधित कर सकें।
