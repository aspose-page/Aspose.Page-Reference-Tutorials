# Aspose.Page का उपयोग करके XPS दस्तावेज़ में टाइल्ड इमेज कैसे जोड़ें

## परिचय

यदि आप यह जानना चाहते हैं **Aspose का उपयोग कैसे करें** ताकि आपके XPS फ़ाइलों में अधिक समृद्ध दृश्य शैली हो, तो आप सही जगह पर आए हैं। इस ट्यूटोरियल में हम Aspose.Page for .NET का उपयोग करके XPS दस्तावेज़ के भीतर **इमेज को टाइल** करने के लिए आवश्यक सटीक चरणों को समझाएंगे। अंत तक आपके पास एक पुन: उपयोग योग्य स्निपेट होगा जिसे आप किसी भी .NET प्रोजेक्ट में ड्रॉप करके ऑन‑द‑फ़्लाई टाइल्ड‑इमेज ग्राफ़िक्स बना सकते हैं।

## त्वरित उत्तर
- **कौन सा लाइब्रेरी चाहिए?** Aspose.Page for .NET  
- **कौन सा मेथड टाइल्ड ब्रश बनाता है?** `CreateImageBrush` के साथ `TileMode = XpsTileMode.Tile`  
- **क्या मैं अपारदर्शिता (opacity) नियंत्रित कर सकता हूँ?** हाँ – `path.Fill.Opacity` सेट करें (उदाहरण: 0.5f)  
- **परीक्षण के लिए लाइसेंस चाहिए?** मूल्यांकन के लिए एक अस्थायी लाइसेंस काम करता है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+

## Aspose.Page क्या है और इमेज को टाइल क्यों करें?

Aspose.Page एक शक्तिशाली API है जो डेवलपर्स को Microsoft Office पर निर्भर हुए बिना XPS, PDF और अन्य पेज‑आधारित फ़ॉर्मेट को जेनरेट, एडिट और रेंडर करने की अनुमति देता है। इमेज को टाइल करना—एक बिटमैप को आकार के भीतर दोहराना—आपको पैटर्न, वॉटरमार्क या बैकग्राउंड टेक्सचर से बड़े क्षेत्रों को भरने में मदद करता है जबकि फ़ाइल आकार कम रहता है।

## Aspose.Page का उपयोग करके XPS दस्तावेज़ में इमेज को टाइल करने का तरीका

नीचे एक पूर्ण, तैयार‑चलाने योग्य उदाहरण दिया गया है। प्रत्येक चरण को साधारण भाषा में समझाया गया है, उसके बाद संबंधित कोड ब्लॉक दिया गया है, ताकि आप देख सकें **क्यों** प्रत्येक पंक्ति महत्वपूर्ण है।

### आवश्यकताएँ

- **Aspose.Page for .NET** – आधिकारिक साइट [से यहाँ](https://reference.aspose.com/page/net/) लाइब्रेरी डाउनलोड करके रेफ़रेंस करें।  
- **डेवलपमेंट एनवायरनमेंट** – Visual Studio (कोई भी एडिशन) या आपका पसंदीदा .NET IDE।

### नेमस्पेस इम्पोर्ट करें

पहले आवश्यक नेमस्पेस को स्कोप में लाएँ ताकि कंपाइलर को XPS क्लासेज़ कहाँ मिलें, पता चल सके।

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

### चरण 1: दस्तावेज़ डायरेक्टरी निर्धारित करें

निर्धारित करें कि जेनरेट की गई XPS फ़ाइल और स्रोत इमेज कहाँ रखी जाएगी। प्लेसहोल्डर को अपने मशीन पर वास्तविक फ़ोल्डर पाथ से बदलें।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

### चरण 2: नया XPS दस्तावेज़ बनाएं

एक खाली XPS दस्तावेज़ इंस्टैंसिएट करें जो टाइल्ड ग्राफ़िक को रखेगा।

```csharp
// Create new XPS Document
XpsDocument doc = new XpsDocument();
```

### चरण 3: टाइल्ड इमेज जोड़ें

यहाँ हम एक आयताकार पाथ बनाते हैं, उसे `ImageBrush` से भरते हैं, और ब्रश को टाइल मोड पर सेट करते हैं। `TileMode` प्रॉपर्टी इंजन को इमेज को क्षैतिज और लंबवत दोनों दिशा में दोहराने के लिए बताती है। अपने परिदृश्य के अनुसार आयत के कॉर्डिनेट्स या स्रोत इमेज को समायोजित करें।

```csharp
// Tile image
// ImageBrush filled rectangle in the right top below
XpsPath path = doc.AddPath(doc.CreatePathGeometry("M 10,160 L 228,160 228,305 10,305"));
path.Fill = doc.CreateImageBrush(dataDir + "R08LN_NN.jpg", new RectangleF(0f, 0f, 128f, 96f), new RectangleF(0f, 0f, 64f, 48f));
((XpsImageBrush)path.Fill).TileMode = XpsTileMode.Tile;
path.Fill.Opacity = 0.5f;
```

### चरण 4: परिणामी XPS दस्तावेज़ सहेजें

अंत में, दस्तावेज़ को डिस्क पर लिखें। आउटपुट फ़ाइल को किसी भी XPS व्यूअर से खोला जा सकता है या आगे Aspose.Page के साथ प्रोसेस किया जा सकता है।

```csharp
// Save resultant XPS document
doc.Save(dataDir + "AddTiledImage_outXPS.xps");
```

## सामान्य समस्याएँ और टिप्स

- **पाथ त्रुटियाँ** – सुनिश्चित करें कि `dataDir` के अंत में स्लैश हो या `Path.Combine` का उपयोग करके सेपरेटर समस्याओं से बचें।  
- **इमेज आकार का मेल न होना** – स्रोत इमेज टाइलिंग क्षेत्र के लिए पर्याप्त बड़ी होनी चाहिए; अन्यथा पैटर्न खिंचा हुआ दिख सकता है।  
- **अपारदर्शिता दिखाई नहीं दे रही** – कुछ व्यूअर XPS पर अपारदर्शिता को अनदेखा करते हैं; ऐसे व्यूअर से टेस्ट करें जो XPS रेंडरिंग को पूरी तरह सपोर्ट करता हो (जैसे Windows पर XPS Viewer)।

## अक्सर पूछे जाने वाले प्रश्न

### Q1: क्या Aspose.Page सभी .NET डेवलपमेंट एनवायरनमेंट्स के साथ संगत है?
A: हाँ, Aspose.Page Visual Studio, Rider, VS Code और किसी भी IDE के साथ काम करता है जो .NET को सपोर्ट करता है।

### Q2: क्या मैं टाइल्ड इमेज की अपारदर्शिता समायोजित कर सकता हूँ?
A: बिल्कुल। उदाहरण में `path.Fill.Opacity = 0.5f;` सेट किया गया है—आप 0 (पारदर्शी) से 1 (अपारदर्शी) के बीच कोई भी फ़्लोट वैल्यू रख सकते हैं।

### Q3: Aspose.Page for .NET में अन्य टाइल मोड उपलब्ध हैं?
A: हाँ। `XpsTileMode.Tile` के अलावा आप `FlipX`, `FlipY`, और `FlipXY` का उपयोग करके मिरर पैटर्न बना सकते हैं।

### Q4: Aspose.Page के अस्थायी लाइसेंस को कैसे संभालूँ?
A: विस्तृत जानकारी और ट्रायल लाइसेंस प्राप्त करने के लिए Aspose वेबसाइट पर [अस्थायी लाइसेंस](https://purchase.aspose.com/temporary-license/) पेज देखें।

### Q5: मैं मदद कहाँ प्राप्त कर सकता हूँ या Aspose.Page समुदाय से कैसे जुड़ूँ?
A: प्रश्न पूछने, स्निपेट साझा करने और अन्य डेवलपर्स से सीखने के लिए [Aspose.Page फ़ोरम](https://forum.aspose.com/c/page/39) पर जाएँ।

### Q6: क्या मैं इस विधि का उपयोग वॉटरमार्क प्रभाव बनाने के लिए कर सकता हूँ?
A: हाँ। अपारदर्शिता को कम करके और अर्ध‑पारदर्शी इमेज चुनकर टाइल्ड ब्रश को दोहराने वाला वॉटरमार्क प्रभाव आसानी से बनाया जा सकता है।

## निष्कर्ष

अब आप जानते हैं **Aspose का उपयोग करके** XPS दस्तावेज़ में टाइल्ड इमेज कैसे जोड़ें, उसकी अपारदर्शिता कैसे नियंत्रित करें, और परिणाम को आगे उपयोग के लिए कैसे सहेजें। यह तकनीक बैकग्राउंड पैटर्न, वॉटरमार्क या किसी भी स्थिति में जहाँ दोहराने वाला ग्राफ़िक दृश्य आकर्षण जोड़ता है बिना फ़ाइल आकार बढ़ाए, के लिए आदर्श है। विभिन्न आकारों, इमेज और टाइल मोड के साथ प्रयोग करने में संकोच न करें ताकि आपका प्रोजेक्ट आवश्यकतानुसार अनुकूलित हो सके।
