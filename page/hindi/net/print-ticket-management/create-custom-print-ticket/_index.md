# टिकट कैसे जोड़ें: Aspose.Page for .NET के साथ कस्टम प्रिंट टिकट बनाएं

## परिचय

यदि आपको .NET एप्लिकेशन में **how to add ticket** कार्यक्षमता की आवश्यकता है, तो Aspose.Page आपको कोड से सीधे कस्टम प्रिंट टिकट उत्पन्न करने की शक्ति देता है। इस ट्यूटोरियल में हम पूरे प्रक्रिया को चरण‑दर‑चरण देखेंगे—पर्यावरण सेटअप, XPS दस्तावेज़ बनाना, कस्टम जॉब प्रिंट टिकट संलग्न करना, और परिणाम को सहेजना। अंत तक, आप किसी भी प्रिंटिंग वर्कफ़्लो में टिकट समर्थन आत्मविश्वास के साथ जोड़ सकेंगे।

## त्वरित उत्तर
- **“add ticket” का क्या अर्थ है?** यह एक कस्टम प्रिंट टिकट (XPS मेटाडेटा) को एम्बेड करने को दर्शाता है जो प्रिंटर सेटिंग्स को नियंत्रित करता है।  
- **कौन सा लाइब्रेरी आवश्यक है?** Aspose.Page for .NET.  
- **क्या मुझे लाइसेंस चाहिए?** मूल्यांकन के लिए एक अस्थायी लाइसेंस काम करता है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **क्या मैं इसे .NET Core के साथ उपयोग कर सकता हूँ?** हाँ, Aspose.Page .NET Framework और .NET Core दोनों को समर्थन देता है।  
- **इम्प्लीमेंटेशन में कितना समय लगेगा?** सामान्यतः बुनियादी टिकट के लिए 15 मिनट से कम।

## कस्टम प्रिंट टिकट क्या है?
एक कस्टम प्रिंट टिकट XML‑आधारित विवरण है जो प्रिंटर प्राथमिकताओं (जैसे कोलेटर, कॉपीज़, कलर मैनेजमेंट आदि) को XPS दस्तावेज़ के साथ ले जाता है। यह डेवलपर्स को प्रोग्रामेटिक रूप से यह निर्धारित करने की अनुमति देता है कि दस्तावेज़ कैसे प्रिंट होना चाहिए, जिससे क्लाइंट साइड पर मैनुअल कॉन्फ़िगरेशन की आवश्यकता समाप्त हो जाती है।

## Aspose.Page के साथ टिकट समर्थन क्यों जोड़ें?
- **सूक्ष्म नियंत्रण:** कोड से कोलेटर, कॉपी संख्या, मीडिया प्रकार आदि सेट करें।  
- **क्रॉस‑प्लेटफ़ॉर्म संगतता:** वही टिकट उन सभी प्रिंटरों पर काम करता है जो XPS मेटाडेटा को समझते हैं।  
- **सहज एकीकरण:** अतिरिक्त सेवाओं के बिना सीधे आपके मौजूदा .NET प्रोजेक्ट्स के साथ काम करता है।

## पूर्वापेक्षाएँ

- C# और .NET विकास में बुनियादी प्रवीणता।  
- Visual Studio (कोई भी नवीनतम संस्करण) स्थापित है।  
- Aspose.Page for .NET लाइब्रेरी आपके प्रोजेक्ट में जोड़ी गई है।  

यदि आपने अभी तक लाइब्रेरी नहीं जोड़ी है, तो आप इसे [Aspose.Page for .NET documentation](https://reference.aspose.com/page/net/) से डाउनलोड कर सकते हैं। समुदाय सहायता के लिए, [Aspose.Page forum](https://forum.aspose.com/c/page/39) पर जाएँ।

## नेमस्पेस आयात करें

XPS और मेटाडेटा क्लासेज़ को उजागर करने वाले नेमस्पेस को आयात करके शुरू करें।

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsMetadata;
using Aspose.Page.XPS.XpsModel;
using System;
using System.Drawing;
```

अब हम इम्प्लीमेंटेशन को चरण‑दर‑चरण समझेंगे।

## चरण 1: दस्तावेज़ डायरेक्टरी सेट करें

निर्धारित करें कि उत्पन्न XPS फ़ाइल कहाँ सहेजी जाएगी।

```csharp
string dir = "Your Document Directory";
```

## चरण 2: नया XPS दस्तावेज़ बनाएं

एक नया XPS दस्तावेज़ बनाएं जो पृष्ठों और टिकट को रखेगा।

```csharp
XpsDocument xDocs = new XpsDocument();
```

## चरण 3: कस्टम जॉब प्रिंट टिकट जोड़ें

दस्तावेज़ में एक कस्टम जॉब प्रिंट टिकट संलग्न करें। यह **how to add ticket** कार्यक्षमता का मूल है—यहाँ आप कोलेटर, कॉपीज़, रेंडरिंग इंटेंट, कलर मैनेजमेंट और अन्य आवश्यक सेटिंग्स निर्दिष्ट करते हैं।

```csharp
xDocs.JobPrintTicket = new JobPrintTicket(
    new PageDevModeSnaphot("SABlAGwAbABvACEAAAA="),
    new DocumentCollate(Collate.CollateOption.Collated),
    // Add other print settings as needed
);
```

> **प्रो टिप:** प्लेसहोल्डर स्नैपशॉट स्ट्रिंग को अपने प्रिंटर की क्षमताओं के अनुरूप Base64‑एन्कोडेड DEVMODE स्ट्रक्चर से बदलें।

## चरण 4: दस्तावेज़ सहेजें

XPS दस्तावेज़ (एम्बेडेड टिकट के साथ) को डिस्क पर सहेजें।

```csharp
xDocs.Save(dir + "output1.xps");
```

जब आप *output1.xps* को ऐसे व्यूअर में खोलते हैं जो XPS मेटाडेटा को मानता है, तो प्रिंटर स्वचालित रूप से टिकट में परिभाषित सेटिंग्स लागू करेगा।

## सामान्य समस्याएँ और समाधान

| Issue | Cause | Fix |
|-------|-------|-----|
| टिकट लागू नहीं हुआ | व्यूअर XPS मेटाडेटा को अनदेखा करता है | ऐसा प्रिंटर ड्राइवर उपयोग करें जो XPS प्रिंट टिकट को समर्थन देता हो या Microsoft XPS Viewer जैसे व्यूअर का उपयोग करें। |

| अनुमतियों की कमी | `dir` पर लिखने की अनुमति अस्वीकृत | सुनिश्चित करें कि एप्लिकेशन पर्याप्त फ़ाइल‑सिस्टम अधिकारों के साथ चल रहा है। |
