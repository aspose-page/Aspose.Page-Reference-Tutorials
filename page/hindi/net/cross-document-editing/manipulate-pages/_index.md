# Aspose.Page for .NET के साथ XPS दस्तावेज़ मर्ज करें

## परिचय

इस ट्यूटोरियल में आप **XPS दस्तावेज़ों को मर्ज** करने और उनके पृष्ठों को Aspose.Page लाइब्रेरी का उपयोग करके .NET वातावरण में कैसे नियंत्रित किया जाए, यह जानेंगे। चाहे आपको कई रिपोर्टों को एक ही XPS फ़ाइल में संयोजित करना हो या परिष्कृत आउटपुट के लिए पृष्ठों को पुनर्व्यवस्थित करना हो, यह गाइड चरण‑दर‑चरण स्पष्ट व्याख्याओं और तैयार‑चलाने‑योग्य कोड के साथ प्रक्रिया को दिखाता है।

## त्वरित उत्तर
- **Aspose.Page से मैं क्या कर सकता हूँ?** XPS दस्तावेज़ मर्ज करना, पृष्ठ जोड़ना, हटाना, और परिणाम सहेजना।  
- **परीक्षण के लिए लाइसेंस चाहिए?** मूल्यांकन के लिए एक अस्थायी लाइसेंस उपलब्ध है।  
- **कौन‑से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+।  
- **क्या Visual Studio आवश्यक है?** नहीं, कोई भी IDE जो C# को सपोर्ट करता है काम करेगा, लेकिन Visual Studio की सलाह दी जाती है।  
- **मर्ज में कितना समय लगता है?** सामान्य‑आकार की XPS फ़ाइलों के लिए कुछ सेकंड।

## XPS दस्तावेज़ों का मर्ज क्या है?
XPS दस्तावेज़ों का मर्ज करने का अर्थ है दो या अधिक मौजूदा XPS फ़ाइलों के पृष्ठों को लेकर उन्हें एक ही XPS दस्तावेज़ में संयोजित करना। यह समेकित रिपोर्ट बनाने, बहुपृष्ठीय मैनुअल संकलित करने, या प्रिंट‑तैयार पैकेज तैयार करने के लिए उपयोगी है, बिना किसी अन्य फ़ॉर्मेट में बदलने की आवश्यकता के।

## .NET के लिए Aspose.Page क्यों उपयोग करें?
Aspose.Page एक **शुद्ध .NET API** प्रदान करता है जो सीधे XPS फ़ाइलों के साथ काम करता है—बाहरी टूल या थर्ड‑पार्टी घटकों की आवश्यकता नहीं। यह पृष्ठ क्रम, सम्मिलन बिंदु, और सामग्री संरक्षण पर सूक्ष्म नियंत्रण देता है, जिससे मर्ज प्रक्रिया विश्वसनीय और तेज़ बनती है।

## पूर्वापेक्षाएँ

- **Aspose.Page for .NET** – डाउनलोड के लिए देखें [Aspose.Page for .NET documentation](https://reference.aspose.com/page/net/).  
- **डेवलपमेंट वातावरण** – Visual Studio, Rider, या कोई भी IDE जो C# को सपोर्ट करता हो।  
- **इनपुट XPS फ़ाइलें** – तीन नमूना फ़ाइलें (`input1.xps`, `input2.xps`, `input3.xps`) को किसी ज्ञात फ़ोल्डर में रखें।

## नेमस्पेस आयात करें

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

इन नेमस्पेस से आपको कोर XPS दस्तावेज़ क्लासेज़, पेज मॉडल, और बेसिक ड्रॉइंग यूटिलिटीज़ तक पहुंच मिलती है।

## चरण 1: दस्तावेज़ डायरेक्टरी सेट करें

```csharp
string dataDir = "Your Document Directory";
```

**Your Document Directory** को उस पूर्ण पथ से बदलें जहाँ आपके XPS फ़ाइलें स्थित हैं, उदाहरण के तौर पर `C:\\Docs\\XpsFiles\\`।

## चरण 2: XPS दस्तावेज़ इंस्टेंस बनाएं

```csharp
XpsDocument doc1 = new XpsDocument(dataDir + "input1.xps");
XpsDocument doc2 = new XpsDocument(dataDir + "input2.xps");
XpsDocument doc3 = new XpsDocument(dataDir + "input3.xps");
XpsDocument doc4 = new XpsDocument();
```

- `doc1`, `doc2`, और `doc3` स्रोत दस्तावेज़ों को दर्शाते हैं जिन्हें आप मर्ज करना चाहते हैं।  
- `doc4` एक खाली XPS दस्तावेज़ है जो मर्ज किए गए परिणाम को रखेगा।

## चरण 3: पृष्ठ सम्मिलित, जोड़ें और हटाएँ

```csharp
doc4.InsertPage(1, doc2.Page, false);
doc4.AddPage(doc3.Page, false);
doc4.RemovePageAt(2);
doc4.InsertPage(2, doc1.SelectActivePage(3), false);
```

यहाँ प्रत्येक पंक्ति क्या करती है:

1. **InsertPage(1, doc2.Page, false)** – `doc2` के पहले पृष्ठ को `doc4` में स्थिति 1 पर रखता है।  
2. **AddPage(doc3.Page, false)** – `doc3` के पहले पृष्ठ को `doc4` के अंत में जोड़ता है।  
3. **RemovePageAt(2)** – अब इंडेक्स 2 पर मौजूद पृष्ठ को हटाता है (अनावश्यक पृष्ठों को हटाने के लिए उपयोगी)।  
4. **InsertPage(2, doc1.SelectActivePage(3), false)** – `doc1` के तीसरे पृष्ठ को स्थिति 2 पर डालता है, जिससे मर्ज पूरा होता है।

इन ऑपरेशनों से आप **XPS दस्तावेज़ों को मर्ज** करते हुए पृष्ठों को पुनः क्रमबद्ध या हटाने की क्षमता प्राप्त करते हैं।

## चरण 4: मर्ज किए गए दस्तावेज़ को सहेजें

```csharp
doc4.Save(dataDir + "out.xps");
```

अंतिम मर्ज किया गया XPS फ़ाइल (`out.xps`) उसी डायरेक्टरी में लिखी जाती है। अब आप इसे किसी भी XPS व्यूअर में खोल सकते हैं या Aspose.Page के साथ आगे प्रोसेस कर सकते हैं।

## सामान्य समस्याएँ और समाधान
- **फ़ाइल नहीं मिली** – `dataDir` पथ को दोबारा जांचें और सुनिश्चित करें कि इनपुट फ़ाइलें मौजूद हैं।  
- **अमान्य पृष्ठ इंडेक्स** – पृष्ठ इंडेक्स 1‑आधारित होते हैं; गैर‑मौजूद पृष्ठ को सम्मिलित करने पर अपवाद उत्पन्न होगा।  
- **लाइसेंस त्रुटियाँ** – उत्पादन में डिप्लॉय करने से पहले अस्थायी या पूर्ण लाइसेंस का उपयोग करें।

## FAQ's

### Q1: क्या मैं विभिन्न XPS दस्तावेज़ों के पृष्ठों को हेरफेर कर सकता हूँ?

A1: हाँ, ट्यूटोरियल में दिखाए अनुसार आप कई XPS दस्तावेज़ों से पृष्ठों को नई दस्तावेज़ में सम्मिलित कर सकते हैं।

### Q2: मैं किसी विशिष्ट पृष्ठ को दस्तावेज़ से कैसे हटाऊँ?

A2: `RemovePageAt` मेथड का उपयोग करें और हटाने वाले पृष्ठ का इंडेक्स निर्दिष्ट करें।

### Q3: क्या Aspose.Page Visual Studio के साथ संगत है?

A3: हाँ, Aspose.Page पूरी तरह से Visual Studio के साथ संगत है, जिससे इसे आपके .NET प्रोजेक्ट में एकीकृत करना आसान हो जाता है।

### Q4: क्या कोई लाइसेंसिंग विकल्प उपलब्ध हैं?

A4: हाँ, आप लाइसेंसिंग विकल्पों का पता लगा सकते हैं और अस्थायी लाइसेंस [यहाँ](https://purchase.aspose.com/temporary-license/) प्राप्त कर सकते हैं।

### Q5: मुझे समर्थन या प्रश्न पूछने के लिए कहाँ जाना चाहिए?

A5: समर्थन प्राप्त करने और समुदाय से जुड़ने के लिए [Aspose.Page forum](https://forum.aspose.com/c/page/39) देखें।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं तीन से अधिक XPS फ़ाइलें मर्ज कर सकता हूँ?**  
उत्तर: बिल्कुल। अतिरिक्त `XpsDocument` इंस्टेंस बनाएं और `InsertPage` या `AddPage` को बार‑बार उपयोग करके बड़े मर्ज्ड दस्तावेज़ तैयार करें।

**प्रश्न: क्या मर्ज मूल फ़ॉर्मेटिंग और ग्राफ़िक्स को बरकरार रखता है?**  
उत्तर: हाँ। Aspose.Page पृष्ठ सामग्री को बाइट‑दर‑बाइट कॉपी करता है, इसलिए टेक्स्ट, इमेज और वेक्टर ग्राफ़िक्स अपरिवर्तित रहते हैं।

**प्रश्न: मैं बिना इंडेक्स निर्दिष्ट किए अंत में पृष्ठ कैसे जोड़ूँ?**  
उत्तर: `AddPage(sourcePage, false)` का उपयोग करें, जो पृष्ठ को दस्तावेज़ के अंत में जोड़ देता है।

**प्रश्न: क्या सर्वर पर UI के बिना XPS दस्तावेज़ मर्ज करना संभव है?**  
उत्तर: API पूरी तरह हेडलेस है; आप इसे ASP.NET, Azure Functions, या किसी भी सर्वर‑साइड .NET वातावरण में चला सकते हैं।

**प्रश्न: यदि मेरी XPS फ़ाइलें पासवर्ड‑सुरक्षित हैं तो क्या होगा?**  
उत्तर: वर्तमान में Aspose.Page एन्क्रिप्टेड XPS फ़ाइलों को सपोर्ट नहीं करता; मर्ज करने से पहले आपको उन्हें डिक्रिप्ट करना होगा।
