# XPS दस्तावेज़ बनाएं – Glyph Clone जोड़ें और रंग बदलें Aspose.Page for .NET के साथ

## परिचय

इस चरण‑दर‑चरण गाइड में आपका स्वागत है जो आपको Aspose.Page for .NET का उपयोग करके **XPS दस्तावेज़** फ़ाइलें बनाने, glyph को क्लोन करने और उनके रंग बदलने का तरीका दिखाता है। चाहे आप डायनामिक रिपोर्ट, इनवॉइस, या कस्टम ग्राफ़िक्स बना रहे हों, इन तकनीकों में महारत हासिल करने से आप .NET इकोसिस्टम से बाहर निकले बिना दृश्य रूप से समृद्ध XPS आउटपुट उत्पन्न कर सकते हैं।

## त्वरित उत्तर
- **मुख्य लक्ष्य क्या है?** XPS दस्तावेज़ बनाना, glyph क्लोन जोड़ना, और उनके रंग बदलना।  
- **कौनसी लाइब्रेरी उपयोग की जाती है?** Aspose.Page for .NET।  
- **क्या मुझे लाइसेंस चाहिए?** मूल्यांकन के लिए एक अस्थायी लाइसेंस उपलब्ध है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **कौनसे .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **परिणाम कैसे सहेजें?** XPS फ़ाइल को डिस्क पर लिखने के लिए `Save` मेथड का उपयोग करें।

## पूर्वापेक्षाएँ

ट्यूटोरियल में डुबने से पहले, सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ हैं:

- C# प्रोग्रामिंग भाषा की बुनियादी समझ।  
- Visual Studio या कोई अन्य पसंदीदा C# विकास पर्यावरण स्थापित हो।  
- Aspose.Page for .NET लाइब्रेरी। आप इसे [here](https://releases.aspose.com/page/net/) से डाउनलोड कर सकते हैं।  
- XPS दस्तावेज़ फ़ॉर्मेट की परिचितता।

## नेमस्पेस आयात करें

शुरू करने के लिए, अपने C# प्रोजेक्ट में आवश्यक नेमस्पेस शामिल करें:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

## XPS दस्तावेज़ कैसे बनाएं और Glyph क्लोन जोड़ें

### चरण 1: अपने दस्तावेज़ डायरेक्टरी सेट करें

सबसे पहले वह डायरेक्टरी सेट करें जहाँ आपके दस्तावेज़ संग्रहीत होंगे:

```csharp
string dataDir = "Your Document Directory";
```

### चरण 2: पहला XPS दस्तावेज़ बनाएं

अब, चलिए पहला XPS दस्तावेज़ बनाते हैं:

```csharp
XpsDocument doc1 = new XpsDocument();
```

### चरण 3: पहले दस्तावेज़ में Glyph जोड़ें

निर्दिष्ट पैरामीटरों का उपयोग करके पहले दस्तावेज़ में glyph जोड़ें:

```csharp
XpsGlyphs glyphs = doc1.AddGlyphs("Times New Roman", 200, FontStyle.Bold, 50, 250, "Test");
```

### चरण 4: Glyph को Solid Color Brush से भरें

पहले दस्तावेज़ में glyph को **solid color brush** से भरें, उदाहरण के लिए, हरा रंग:

```csharp
glyphs.Fill = doc1.CreateSolidColorBrush(Color.Green);
```

### चरण 5: दूसरा XPS दस्तावेज़ बनाएं

अब, दूसरा XPS दस्तावेज़ बनाएं:

```csharp
XpsDocument doc2 = new XpsDocument();
```

### चरण 6: Glyph Clone को दूसरे दस्तावेज़ में कैसे जोड़ें

पहले दस्तावेज़ से glyph को क्लोन करें और उन्हें दूसरे दस्तावेज़ में जोड़ें:

```csharp
glyphs = doc2.Add(glyphs.Clone());
```

### चरण 7: क्लोन किए गए Glyph का रंग कैसे बदलें

दूसरे दस्तावेज़ में क्लोन किए गए glyph का रंग बदलें, उदाहरण के लिए, लाल। यह क्लोन करने के बाद glyph का **रंग कैसे बदलें** यह दर्शाता है:

```csharp
((XpsSolidColorBrush)glyphs.Fill).Color = doc2.CreateColor(Color.Red);
```

### चरण 8: XPS फ़ाइल सहेजें – पहला दस्तावेज़

पहले XPS दस्तावेज़ को उस फ़ोल्डर में सहेजें जिसे आपने पहले परिभाषित किया था:

```csharp
doc1.Save(dataDir + "out1.xps");
```

### चरण 9: XPS फ़ाइल सहेजें – दूसरा दस्तावेज़

दूसरे XPS दस्तावेज़ को सहेजें:

```csharp
doc2.Save(dataDir + "out2.xps");
```

बधाई हो! आपने सफलतापूर्वक **XPS दस्तावेज़** फ़ाइलें बनाई, glyph क्लोन जोड़े, और उनके रंग बदलें Aspose.Page for .NET का उपयोग करके।

## Glyph क्लोनिंग और रंग परिवर्तन का उपयोग क्यों करें?

- **पुन: उपयोगिता:** मौजूदा glyph को क्लोन करके जटिल वेक्टर आकारों को पुनः परिभाषित किए बिना पुन: उपयोग करें।  
- **प्रदर्शन:** शून्य से glyph को पुनः बनाने की तुलना में प्रोसेसिंग समय कम करता है।  
- **डिज़ाइन लचीलापन:** एक ही glyph पर विभिन्न `SolidColorBrush` इंस्टेंस लागू करके विभिन्न दृश्य थीम प्राप्त करें।  
- **क्रॉस‑डॉक्यूमेंट संपादन:** कई XPS फ़ाइलों में glyph को क्लोन करें, जिससे रिपोर्ट में सुसंगत ब्रांडिंग संभव हो।

## सामान्य समस्याएँ और ट्रबलशूटिंग

| समस्या | समाधान |
|-------|----------|
| **Clone null लौटाता है** | सुनिश्चित करें कि स्रोत glyph (`glyphs`) को क्लोन करने से पहले पहले दस्तावेज़ में जोड़ा गया है। |
| **रंग नहीं बदलता** | `glyphs.Fill` को `XpsSolidColorBrush` में कास्ट करें इससे पहले कि `Color` प्रॉपर्टी सेट करें (जैसा कि चरण 7 में दिखाया गया है)। |
| **फ़ाइल सहेजी नहीं गई** | `dataDir` यह सुनिश्चित करें कि यह एक वैध, लिखने योग्य फ़ोल्डर की ओर इशारा करता है और आपके पास उचित फ़ाइल‑सिस्टम अनुमतियाँ हैं। |
| **अप्रत्याशित glyph आकार** | `AddGlyphs` में दूसरे तर्क (फ़ॉन्ट आकार पैरामीटर) को समायोजित करके glyph को उचित रूप से स्केल करें। |
