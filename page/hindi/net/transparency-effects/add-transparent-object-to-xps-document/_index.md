# XPS को PDF में निर्यात करें – Aspose.Page के साथ पारदर्शी ऑब्जेक्ट जोड़ें

## परिचय

इस ट्यूटोरियल में आप सीखेंगे कि XPS दस्तावेज़ में पारदर्शी ऑब्जेक्ट कैसे जोड़ें और फिर Aspose.Page for .NET के साथ **XPS को PDF में निर्यात** करें। पारदर्शिता आपके दस्तावेज़ों को अधिक आधुनिक दिखा सकती है और महत्वपूर्ण जानकारी को उजागर करने में मदद करती है। हम प्रत्येक चरण को विस्तार से समझाएंगे, कोड के पीछे की तर्क को बताएँगे, और XPS फ़ाइल को PDF में बदलकर आसान शेयरिंग के लिए वर्कफ़्लो को कैसे समाप्त करें, यह दिखाएँगे।

## त्वरित उत्तर
- **XPS को PDF में निर्यात** का क्या अर्थ है? XPS फ़ाइल को PDF फ़ाइल में बदलना, जबकि लेआउट, रंग और पारदर्शिता को संरक्षित रखा जाता है।  
- **पहले पारदर्शिता क्यों जोड़ें?** पारदर्शी ऑब्जेक्ट आपको लेयरड ग्राफ़िक्स बनाने देते हैं जो अंतिम PDF में शानदार दिखते हैं।  
- **क्या मुझे लाइसेंस चाहिए?** हाँ, उत्पादन उपयोग के लिए एक वैध Aspose.Page लाइसेंस आवश्यक है।  
- **क्या यह .NET Core पर चल सकता है?** बिल्कुल – Aspose.Page .NET Core, .NET 5/6 और पूर्ण .NET Framework को सपोर्ट करता है।  
- **और उदाहरण कहाँ मिलेंगे?** नीचे दिए गए Aspose.Page दस्तावेज़ और कम्युनिटी फ़ोरम देखें।

## पूर्वापेक्षाएँ

ट्यूटोरियल शुरू करने से पहले, सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ मौजूद हैं:

- Aspose.Page for .NET: सुनिश्चित करें कि आपके पास Aspose.Page लाइब्रेरी for .NET स्थापित है। आप इसे [Aspose.Page for .NET Documentation](https://reference.aspose.com/page/net/) से डाउनलोड कर सकते हैं।

## नेमस्पेस इम्पोर्ट करें

शुरू करने के लिए, अपने प्रोजेक्ट में आवश्यक नेमस्पेस शामिल करें:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

अब, चलिए चरण‑दर‑चरण गाइड के साथ आगे बढ़ते हैं।

## चरण 1: नया XPS दस्तावेज़ बनाएं

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";
// Create new XPS Document
XpsDocument doc = new XpsDocument();
```

यह कोड Aspose.Page for .NET का उपयोग करके एक नया XPS दस्तावेज़ इनिशियलाइज़ करता है।

## चरण 2: पारदर्शिता प्रदर्शित करें

```csharp
// Just to demonstrate transparency
doc.AddPath(doc.CreatePathGeometry("M120,0 H400 v1000 H120")).Fill = doc.CreateSolidColorBrush(Color.Gray);
doc.AddPath(doc.CreatePathGeometry("M300,120 h600 V420 h-600")).Fill = doc.CreateSolidColorBrush(Color.Gray);
```

इन पंक्तियों से दो ग्रे पाथ बनते हैं जो बाद में जोड़े जाने वाले पारदर्शी आकारों के लिए बैकग्राउंड के रूप में काम करेंगे।

## चरण 3: बंद आयत ज्यामिति के साथ पाथ बनाएं

```csharp
XpsPath path1 = doc.CreatePath(doc.CreatePathGeometry("M20,20 h200 v200 h-200 z"));
path1.Fill = doc.CreateSolidColorBrush(Color.Blue);
```

यहाँ हम एक नीला आयत बनाते हैं। क्योंकि हम बाद में इसकी अपारदर्शिता बदलेंगे, आयत अंतिम PDF में अर्ध‑पारदर्शी दिखेगा।

## चरण 4: पाथ और रंगों को बदलें

```csharp
XpsPath path2 = doc.Add(path1);
path2.Fill = doc.CreateSolidColorBrush(Color.Green);
```

हम पहले आयत की क्लोन बनाते हैं, उसे पेज में जोड़ते हैं, और उसका फ़िल रंग हरा बदलते हैं। यह दर्शाता है कि मौजूदा ज्यामिति को पुन: उपयोग करना कितना आसान है।

## चरण 5: पाथ को क्लोन और ट्रांसफ़ॉर्म करें

```csharp
XpsPath path3 = doc.Add(path2);
path3.RenderTransform = doc.CreateMatrix(1, 0, 0, 1, 0, 300);
path3.Fill = doc.CreateSolidColorBrush(Color.Red);
```

क्लोन किया गया पाथ 300 यूनिट नीचे शिफ्ट किया गया है और लाल रंग से रंगा गया है, जिससे आपको एक स्टैक्ड‑लेयर प्रभाव मिलता है।

## चरण 6: पाथ को दोहराएँ और संशोधित करें

```csharp
XpsPath path4 = doc.AddPath(path2.Data);
path4.RenderTransform = doc.CreateMatrix(1, 0, 0, 1, 300, 0);
path4.Fill = doc.CreateSolidColorBrush(Color.Blue);
```

हम `path2` से ज्यामिति डेटा को पुन: उपयोग करते हैं, उसे दाएँ की ओर ले जाते हैं, और फिर से नीला फ़िल लागू करते हैं।

## चरण 7: अपारदर्शिता प्रबंधित करें

```csharp
XpsPath path5 = doc.Add(path4);
path5.RenderTransform = path5.RenderTransform.Clone();
path5.RenderTransform.Translate(0, 300);
path5.Fill.Opacity = 0.8f;
```

`Opacity` प्रॉपर्टी को 0.8 सेट करने से यह आयत 80 % अपारदर्शी हो जाता है, जो दर्शाता है कि आप प्रत्येक तत्व के लिए पारदर्शिता को कैसे सूक्ष्म रूप से समायोजित कर सकते हैं।

## चरण 8: XPS दस्तावेज़ सहेजें

```csharp
doc.Save(dataDir + "WorkingWithTransparency_out.xps");
```

XPS फ़ाइल अब सभी पारदर्शी ऑब्जेक्ट लागू करके सहेजी गई है।  

**PDF में निर्यात:** **XPS को PDF में निर्यात** करने के लिए, बस `doc.Save` को फिर से `.pdf` एक्सटेंशन के साथ कॉल करें, उदाहरण के लिए, `doc.Save(dataDir + "TransparentDocument.pdf");`। समान दृश्य गुणवत्ता, जिसमें अपारदर्शिता सेटिंग्स शामिल हैं, परिणामी PDF में संरक्षित रहती है।

## पारदर्शिता जोड़ने के बाद XPS को PDF में निर्यात क्यों करें?

- **सार्वभौमिक संगतता:** PDF विभिन्न प्लेटफ़ॉर्म पर व्यापक रूप से समर्थित है, जबकि XPS अधिक विशिष्ट है।  
- **संरक्षित दृश्य प्रभाव:** Aspose.Page रूपांतरण के दौरान पारदर्शिता, ग्रेडिएंट और मैट्रिक्स ट्रांसफ़ॉर्म को अपरिवर्तित रखता है।  
- **आसान शेयरिंग:** PDF को ब्राउज़र, मोबाइल डिवाइस और कई थर्ड‑पार्टी रीडर्स में अतिरिक्त प्लगइन्स के बिना खोला जा सकता है।

## सामान्य उपयोग केस

- **मार्केटिंग ब्रोशर** जहाँ लेयरड ग्राफ़िक्स प्रीमियम फील देते हैं।  
- **तकनीकी आरेख** जिन्हें लेआउट को अव्यवस्थित किए बिना हाइलाइटेड सेक्शन चाहिए।  
- **रिपोर्ट जनरेशन** जहाँ आप वॉटरमार्क या लोगो को आंशिक अपारदर्शिता के साथ ओवरले करना चाहते हैं।

## टिप्स और सावधानियाँ

- **प्रो टिप:** हमेशा `Opacity` मान को 0 और 1 के बीच सेट करें। इस रेंज के बाहर के मान क्लैंप हो जाते हैं और अप्रत्याशित परिणाम दे सकते हैं।  
- **परफ़ॉर्मेंस टिप:** ज्यामिति (`path2.Data`) को पुन: उपयोग करने से कई समान आकारों की आवश्यकता होने पर मेमोरी उपयोग कम होता है।  
- **पिटफ़ॉल:** पारदर्शिता जोड़ने के बाद सीधे PDF में सहेजना बॉक्स से बाहर काम करता है, लेकिन यदि आप बाद में PDF को अन्य टूल्स से संपादित करते हैं, तो कुछ पुराने व्यूअर्स अपारदर्शिता को सही से रेंडर नहीं कर सकते।

## निष्कर्ष

Aspose.Page for .NET का उपयोग करके XPS दस्तावेज़ों में पारदर्शी ऑब्जेक्ट जोड़ना दृश्य प्रस्तुतियों को बेहतर बनाने का एक बहुमुखी तरीका प्रदान करता है। एक बार आपका XPS फ़ाइल इच्छित रूप में दिखे, आप **XPS को PDF में निर्यात** एक ही कोड लाइन में कर सकते हैं, जिससे सभी पारदर्शिता प्रभाव व्यापक वितरण के लिए संरक्षित रहते हैं।

## अक्सर पूछे जाने वाले प्रश्न

### प्रश्न 1: क्या मैं XPS दस्तावेज़ में किसी भी ऑब्जेक्ट पर पारदर्शिता लागू कर सकता हूँ?
A1: हाँ, पारदर्शिता XPS दस्तावेज़ में पाथ, आकार और छवियों जैसे विभिन्न ऑब्जेक्ट्स पर लागू की जा सकती है।

### प्रश्न 2: मैं किसी विशिष्ट तत्व की अपारदर्शिता कैसे समायोजित कर सकता हूँ?
A2: आप Fill या Stroke की opacity प्रॉपर्टी सेट करके किसी विशिष्ट तत्व की पारदर्शिता को समायोजित कर सकते हैं।

### प्रश्न 3: क्या Aspose.Page .NET Core के साथ संगत है?
A3: हाँ, Aspose.Page .NET Core को सपोर्ट करता है, जिससे क्रॉस‑प्लेटफ़ॉर्म विकास संभव होता है।

### प्रश्न 4: क्या मैं Aspose.Page का उपयोग करके XPS दस्तावेज़ों को अन्य फ़ॉर्मेट में निर्यात कर सकता हूँ?
A4: Aspose.Page XPS दस्तावेज़ों को विभिन्न फ़ॉर्मेट, जैसे PDF और इमेजेज़, में निर्यात करने की कार्यक्षमता प्रदान करता है।

### प्रश्न 5: अतिरिक्त समर्थन और कम्युनिटी चर्चा कहाँ मिल सकती है?
A5: अतिरिक्त समर्थन और कम्युनिटी चर्चाओं के लिए, [Aspose.Page Forum](https://forum.aspose.com/c/page/39) पर जाएँ।

### प्रश्न 6: क्या PDF रूपांतरण मेरे पारदर्शिता सेटिंग्स को रखता है?
A6: बिल्कुल। जब आप Aspose.Page के साथ XPS को PDF में निर्यात करते हैं, तो सभी opacity और blend सेटिंग्स बरकरार रहती हैं।

### प्रश्न 7: क्या मैं कई XPS फ़ाइलों को बैच‑प्रोसेस करके PDF में बदल सकता हूँ?
A7: हाँ, आप XPS फ़ाइलों के संग्रह पर लूप करके प्रत्येक के लिए `doc.Save(...".pdf")` कॉल कर सकते हैं, जिससे बैच रूपांतरण स्वचालित हो जाता है।
