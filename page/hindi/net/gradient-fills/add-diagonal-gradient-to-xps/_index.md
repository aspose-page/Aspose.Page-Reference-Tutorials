# Aspose.Page for .NET के साथ डायगोनल ग्रेडिएंट XPS बनाएं

## परिचय

यदि आपको ऐसी **डायगोनल ग्रेडिएंट XPS** फ़ाइलें बनानी हैं जो नज़र को आकर्षित करें, तो Aspose.Page for .NET इसे सरल बनाता है। इस ट्यूटोरियल में आप Aspose.Page API का उपयोग करके XPS दस्तावेज़ में डायगोनल ग्रेडिएंट जोड़ना चरण‑दर‑चरण सीखेंगे। अंत तक आपके पास एक पुन: उपयोग योग्य पैटर्न होगा जिसे आप किसी भी आकार या रंग योजना के अनुसार अनुकूलित कर सकते हैं।

## त्वरित उत्तर
- **API मेथड क्या करता है?** यह एक लीनियर ग्रेडिएंट ब्रश बनाता है जो पाथ के पार डायगोनली फैला होता है।  
- **कौन सा क्लास XPS दस्तावेज़ को दर्शाता है?** `XpsDocument`।  
- **क्या सैंपल चलाने के लिए लाइसेंस चाहिए?** विकास के लिए मुफ्त ट्रायल चल सकता है; उत्पादन के लिए लाइसेंस आवश्यक है।  
- **क्या मैं ग्रेडिएंट की दिशा बदल सकता हूँ?** हाँ—`PointF` के प्रारंभ और अंत मानों को समायोजित करें।  
- **कौन‑से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7।

## **create diagonal gradient xps** क्या है?
डायगोनल ग्रेडिएंट एक स्मूथ रंग परिवर्तन है जो आकार के एक कोने से opposite कोने तक चलता है। XPS में, यह प्रभाव `LinearGradientBrush` को पाथ की fill प्रॉपर्टी पर लागू करके प्राप्त किया जाता है। Aspose.Page लाइब्रेरी लो‑लेवल XPS मार्कअप को एब्स्ट्रैक्ट करती है, जिससे आप रंग और ज्योमेट्री पर ध्यान केंद्रित कर सकते हैं।

## Aspose.Page के साथ डायगोनल ग्रेडिएंट क्यों उपयोग करें?
- **उच्च‑सटीकता रेंडरिंग** – आउटपुट XPS स्पेसिफिकेशन के बिल्कुल अनुरूप होता है।  
- **पूर्ण .NET इंटीग्रेशन** – C#, VB.NET और किसी भी .NET भाषा के साथ काम करता है।  
- **कोई बाहरी निर्भरताएँ नहीं** – सब कुछ इन‑प्रोसेस संभाला जाता है, कोई COM या Office आवश्यक नहीं।  
- **जटिल दस्तावेज़ों के लिए स्केलेबल** – आप एक ही पृष्ठ पर कई ग्रेडिएंट, इमेज और टेक्स्ट को संयोजित कर सकते हैं।

## आवश्यकताएँ

1. **Aspose.Page for .NET लाइब्रेरी** – इसे [यहाँ](https://releases.aspose.com/page/net/) से डाउनलोड करें।  
2. **डेवलपमेंट एनवायरनमेंट** – Visual Studio, Rider, या कोई भी एडिटर जो .NET प्रोजेक्ट्स को सपोर्ट करता हो।  

अब, कोड में डुबकी लगाते हैं।

## नेमस्पेस इम्पोर्ट करें

अपने .NET प्रोजेक्ट में, Aspose.Page लाइब्रेरी से आवश्यक नेमस्पेस शामिल करें ताकि आवश्यक क्लास और मेथड्स तक पहुंच सकें। कोड की शुरुआत में निम्नलिखित नेमस्पेस जोड़ें:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Collections.Generic;
using System.Drawing;
```

## चरण 1: दस्तावेज़ डायरेक्टरी सेट करें

सबसे पहले अपने दस्तावेज़ डायरेक्टरी का पाथ निर्दिष्ट करें। यही वह स्थान होगा जहाँ डायगोनल ग्रेडिएंट वाला परिणामी XPS दस्तावेज़ सहेजा जाएगा।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

## चरण 2: नया XPS दस्तावेज़ बनाएं

Aspose.Page लाइब्रेरी का उपयोग करके एक नया `XpsDocument` प्रारंभ करें।

```csharp
XpsDocument doc = new XpsDocument();
```

## चरण 3: ग्रेडिएंट रंग निर्धारित करें

`XpsGradientStop` ऑब्जेक्ट्स की एक सूची बनाएं, जहाँ प्रत्येक डायगोनल ग्रेडिएंट में एक रंग का प्रतिनिधित्व करता है।

```csharp
List<XpsGradientStop> stops = new List<XpsGradientStop>();
stops.Add(doc.CreateGradientStop(doc.CreateColor(0, 142, 4), 0f));
// ... Repeat for other colors
stops.Add(doc.CreateGradientStop(doc.CreateColor(0, 199, 80), 1f));
```

## चरण 4: पाथ में डायगोनल ग्रेडिएंट जोड़ें

परिभाषित ज्योमेट्री के साथ एक नया पाथ बनाएं, और उस पर डायगोनल ग्रेडिएंट लागू करें। आवश्यकतानुसार रेंडरिंग ट्रांसफ़ॉर्म और fill प्रॉपर्टी को समायोजित करें।

```csharp
XpsPath path = doc.AddPath(doc.CreatePathGeometry("M 10,10 L 228,10 228,100 10,100"));
path.RenderTransform = doc.CreateMatrix(1f, 0f, 0f, 1f, 20f, 70f);
path.Fill = doc.CreateLinearGradientBrush(new PointF(10f, 10f), new PointF(228f, 100f));
((XpsGradientBrush)path.Fill).GradientStops.AddRange(stops);
```

## चरण 5: परिणामी XPS दस्तावेज़ सहेजें

अंत में, संशोधित XPS दस्तावेज़ को निर्दिष्ट डायरेक्टरी में सहेजें।

```csharp
doc.Save(dataDir + "AddDiagonalGradient_outXPS.xps");
```

आपने अब सफलतापूर्वक **डायगोनल ग्रेडिएंट XPS** फ़ाइल बना ली है। विभिन्न कलर स्टॉप्स, ज्योमेट्री स्ट्रिंग्स या ट्रांसफ़ॉर्म मैट्रिक्स के साथ प्रयोग करके विविध दृश्य प्रभाव प्राप्त कर सकते हैं।

## सामान्य समस्याएँ और समाधान
- **ग्रेडिएंट दिखाई नहीं दे रहा** – सुनिश्चित करें कि पाथ ज्योमेट्री बंद है और ब्रश के प्रारंभ/अंत बिंदु पाथ की सीमा के भीतर हैं।  
- **रंग गलत हैं** – `CreateColor` को सही ARGB मानों के साथ उपयोग करें; मेथड 0‑255 रेंज में मानों की अपेक्षा करता है।  
- **फ़ाइल सहेजी नहीं गई** – जांचें कि `dataDir` मौजूदा फ़ोल्डर की ओर इशारा कर रहा है और एप्लिकेशन के पास लिखने की अनुमति है।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं दस्तावेज़ के विभिन्न भागों पर कई ग्रेडिएंट लागू कर सकता हूँ?**  
उत्तर: हाँ, आप कई पाथ बना सकते हैं और प्रत्येक पर अलग‑अलग ग्रेडिएंट लागू कर सकते हैं।

**प्रश्न: क्या पूर्वनिर्धारित ग्रेडिएंट स्टाइल उपलब्ध हैं?**  
उत्तर: Aspose.Page कस्टम ग्रेडिएंट की अनुमति देता है, जिससे आप रंग परिवर्तन पर पूर्ण नियंत्रण रख सकते हैं।

**प्रश्न: क्या Aspose.Page for .NET को अन्य दस्तावेज़ फ़ॉर्मेट्स के साथ उपयोग किया जा सकता है?**  
उत्तर: Aspose.Page मुख्यतः XPS दस्तावेज़ हेरफेर पर केंद्रित है।

**प्रश्न: दस्तावेज़ प्रोसेसिंग से संबंधित त्रुटियों को कैसे संभालें?**  
उत्तर: त्रुटि हैंडलिंग के सर्वोत्तम अभ्यासों के लिए [डॉक्यूमेंटेशन](https://reference.aspose.com/page/net/) देखें।

**प्रश्न: खरीदने से पहले कोई ट्रायल संस्करण उपलब्ध है?**  
उत्तर: हाँ, आप [फ़्री ट्रायल](https://releases.aspose.com/) का उपयोग करके Aspose.Page for .NET का अनुभव ले सकते हैं।

**प्रश्न: ग्रेडिएंट दिशा को वर्टिकल या हॉरिज़ॉन्टल कैसे बदलें?**  
उत्तर: `CreateLinearGradientBrush` में `PointF` आर्ग्यूमेंट्स को बदलकर नए प्रारंभ और अंत बिंदु सेट करें।

**प्रश्न: क्या लाइब्रेरी ग्रेडिएंट में ट्रांसपैरेंसी सपोर्ट करती है?**  
उत्तर: हाँ—`CreateColor` के साथ अल्फा वैल्यू शामिल करके ट्रांसपैरेंट ग्रेडिएंट बना सकते हैं।

## निष्कर्ष

Aspose.Page for .NET XPS दस्तावेज़ों में डायगोनल ग्रेडिएंट जोड़ने की प्रक्रिया को सरल बनाता है। इस गाइड में हमने प्री‑रिक्विज़िट सेटअप से लेकर अंतिम फ़ाइल सहेजने तक सभी चरणों को कवर किया। विभिन्न आकारों और रंग पैलेट्स के साथ प्रयोग जारी रखें ताकि आपके XPS रिपोर्ट, ब्रोशर या इनवॉइस वास्तव में अलग दिखें।
