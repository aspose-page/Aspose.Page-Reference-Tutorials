# Aspose.Page for .NET के साथ XPS दस्तावेज़ संशोधित करें

## परिचय

Aspose.Page for .NET के साथ **XPS दस्तावेज़ फ़ाइलों को कैसे संशोधित करें** इस चरण‑दर‑चरण गाइड में आपका स्वागत है। चाहे आपको हस्ताक्षर डालना हो, वॉटरमार्क जोड़ना हो, या केवल पृष्ठ पर कस्टम टेक्स्ट रखना हो, यह ट्यूटोरियल आपको कुछ ही मिनटों में XPS दस्तावेज़ में **टेक्स्ट कैसे जोड़ें** दिखाता है। हम हर कोड लाइन को समझेंगे, प्रत्येक चरण का महत्व बताएँगे, और सामान्य त्रुटियों से बचने के लिए टिप्स देंगे।

### त्वरित उत्तर
- **यह ट्यूटोरियल क्या कवर करता है?** XPS फ़ाइल के चयनित पृष्ठों पर हस्ताक्षर टेक्स्ट ("Confirmed") जोड़ना।  
- **कौन सी लाइब्रेरी आवश्यक है?** Aspose.Page for .NET (नवीनतम संस्करण)।  
- **क्या लाइसेंस चाहिए?** परीक्षण के लिए एक अस्थायी लाइसेंस काम करता है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+।  
- **इम्प्लीमेंटेशन में कितना समय लगेगा?** बुनियादी हस्ताक्षर सम्मिलन के लिए लगभग 10 मिनट।

## XPS दस्तावेज़ को संशोधित करना क्या है?

XPS (XML Paper Specification) माइक्रोसॉफ्ट का फिक्स्ड‑लेआउट दस्तावेज़ फ़ॉर्मेट है, जो PDF के समान है। XPS दस्तावेज़ को संशोधित करने का अर्थ है प्रोग्रामेटिक रूप से उसकी दृश्य सामग्री बदलना—टेक्स्ट, इमेज या शैप्स जोड़ना—बिना फ़ाइल को किसी अन्य फ़ॉर्मेट में परिवर्तित किए। Aspose.Page एक समृद्ध ऑब्जेक्ट मॉडल प्रदान करता है जिससे आप अपने .NET कोड से सीधे XPS फ़ाइलें संपादित कर सकते हैं।

## XPS दस्तावेज़ संशोधित करने के लिए Aspose.Page क्यों उपयोग करें?

* **पूर्ण नियंत्रण** – पृष्ठों, ग्लिफ़्स, ब्रशेज़ और ट्रांसफ़ॉर्म्स के साथ लो‑लेवल काम।  
* **कोई बाहरी निर्भरताएँ नहीं** – शुद्ध .NET लाइब्रेरी, Office या Adobe घटकों की आवश्यकता नहीं।  
* **क्रॉस‑प्लेटफ़ॉर्म** – .NET Core के माध्यम से Windows, Linux और macOS पर चलता है।  
* **मजबूत प्रदर्शन** – बड़े दस्तावेज़ों को कुशलता से संभालता है और उन्नत टाइपोग्राफी का समर्थन करता है।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हों:

- **Aspose.Page for .NET** – NuGet पैकेज इंस्टॉल करें या आधिकारिक दस्तावेज़ **[यहाँ](https://reference.aspose.com/page/net/)** से लाइब्रेरी डाउनलोड करें।  
- **इनपुट XPS फ़ाइल** – एक नमूना XPS दस्तावेज़ (जैसे `input1.xps`) **[Aspose रिलीज़ पेज़](https://releases.aspose.com/page/net/)** से प्राप्त करें।  
- **वर्किंग डायरेक्टरी** – अपने मशीन पर एक फ़ोल्डर बनाएं जहाँ इनपुट और आउटपुट फ़ाइलें रखी जाएँ और उसका पूर्ण पथ नोट करें; आप इस पथ को कोड में `dir` वेरिएबल को असाइन करेंगे।  
- **डेवलपमेंट एनवायरनमेंट** – Visual Studio 2019/2022, .NET Framework 4.7.2 या बाद का संस्करण, या कोई भी .NET Core/5/6 प्रोजेक्ट।

अब जब सब सेट है, चलिए कोड में डुबकी लगाते हैं।

## नेमस्पेस इम्पोर्ट करें

अपने .NET प्रोजेक्ट में Aspose.Page के लिए आवश्यक नेमस्पेस इम्पोर्ट करके शुरू करें:

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
using System.IO;
```

## चरण 1: XPS दस्तावेज़ स्ट्रीम खोलें

हम स्रोत XPS फ़ाइल को एक स्ट्रीम के रूप में खोलेंगे और एक `XpsDocument` ऑब्जेक्ट बनाएँगे जो पूरे दस्तावेज़ का प्रतिनिधित्व करता है।

```csharp
// ExStart:3
// The path to the documents directory.
string dir = "Your Document Directory";
// Open a stream of XPS file
using (FileStream xpsStream = File.Open(dir + "input1.xps", FileMode.Open, FileAccess.Read))
{
    // Create PS document from stream
    XpsDocument document = new XpsDocument(xpsStream, new XpsLoadOptions());
    // Continue to the next step...
}
// ExEnd:3
```

*प्रो टिप:* फ़ाइल हैंडल को स्वचालित रूप से रिलीज़ करने के लिए स्ट्रीम को `using` ब्लॉक में रैप करें।

## चरण 2: हस्ताक्षर टेक्स्ट बनाएं

अगला, हम एक सॉलिड‑कलर ब्रश बनाएँगे जिसका उपयोग हस्ताक्षर ग्लिफ़्स को पेंट करने के लिए किया जाएगा।

```csharp
// ExStart:4
// Create fill of the signature text
XpsSolidColorBrush textFill = document.CreateSolidColorBrush(Color.BlueViolet);
// Continue to the next step...
// ExEnd:4
```

आप `Color.BlueViolet` को किसी भी `System.Drawing.Color` में बदल सकते हैं जो आपके ब्रांडिंग से मेल खाता हो।

## चरण 3: पृष्ठ निर्धारित करें और हस्ताक्षर जोड़ें

हम उन पृष्ठों को निर्दिष्ट करेंगे जिन्हें हस्ताक्षर चाहिए और फिर प्रत्येक पृष्ठ पर ग्लिफ़्स जोड़ेंगे।

```csharp
// ExStart:5
// Define pages where signature will be set
int[] pageNumbers = new int[] {1, 2, 3};

// For every defined page set signature "Confirmed" at coordinates x=650 and y=950
for (int i = 0; i < pageNumbers.Length; i++)
{
    // Define active page
    document.SelectActivePage(pageNumbers[i]);

    // Create glyphs object
    XpsGlyphs glyphs = document.AddGlyphs("Arial", 24, FontStyle.Bold, 650, 900, "Confirmed");

    // Define fill for glyphs
    glyphs.Fill = textFill;
}
// Continue to the next step...
// ExEnd:5
```

*इन निर्देशांक क्यों?* X और Y मान पॉइंट्स (1/72 इंच) में मापे जाते हैं। इन्हें समायोजित करके आप टेक्स्ट को पृष्ठ लेआउट में ठीक उसी जगह रख सकते हैं जहाँ आपको चाहिए।

## चरण 4: XPS दस्तावेज़ में बदलाव सहेजें

अंत में, संशोधित दस्तावेज़ को डिस्क पर वापस लिखें।

```csharp
// ExStart:6
// Save changed XPS document
document.Save(dir + "input1_out.xps");
// ExEnd:6
```

नया फ़ाइल `input1_out.xps` अब पृष्ठ 1‑3 पर “Confirmed” हस्ताक्षर रखता है।

## सामान्य समस्याएँ और समाधान

| समस्या | कारण | समाधान |
|-------|-------|----------|
| **हस्ताक्षर दिखाई नहीं दे रहा** | गलत निर्देशांक या पृष्ठ चयन नहीं हुआ | प्रत्येक पृष्ठ के लिए `SelectActivePage` कॉल किया गया है यह सुनिश्चित करें और X/Y मान समायोजित करें। |
| **`AddGlyphs` पर एक्सेप्शन** | सर्वर पर फ़ॉन्ट स्थापित नहीं है | निर्दिष्ट फ़ॉन्ट (जैसे Arial) उपलब्ध है यह सुनिश्चित करें, या `document.AddFont` से कस्टम फ़ॉन्ट एम्बेड करें। |
| **आउटपुट फ़ाइल भ्रष्ट** | स्ट्रीम सही से बंद नहीं हुई | सभी स्ट्रीम के लिए `using` स्टेटमेंट उपयोग करें और आवश्यक होने पर `document.Dispose()` कॉल करें। |
| **बड़े फ़ाइलों पर प्रदर्शन धीमा** | पूरे दस्तावेज़ को मेमोरी में लोड करना | पृष्ठों को बैच में प्रोसेस करें या यदि नए संस्करण में उपलब्ध हो तो `XpsLoadOptions` के साथ स्ट्रीमिंग विकल्प उपयोग करें। |

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या Aspose.Page नवीनतम .NET फ्रेमवर्क के साथ संगत है?**  
**उत्तर:** हाँ, Aspose.Page नियमित रूप से अपडेट किया जाता है ताकि .NET Framework 4.5+, .NET Core 3.1+, .NET 5 और .NET 6 को सपोर्ट कर सके।

**प्रश्न: क्या मैं जोड़े गए टेक्स्ट का फ़ॉन्ट और स्टाइल कस्टमाइज़ कर सकता हूँ?**  
**उत्तर:** बिल्कुल। `AddGlyphs` के पैरामीटर (फ़ॉन्ट नाम, आकार, `FontStyle`) को बदलें ताकि आपका डिज़ाइन फिट हो।

**प्रश्न: XPS फ़ाइलों के आकार पर कोई सीमा है क्या?**  
**उत्तर:** Aspose.Page बड़े दस्तावेज़ों को संभाल सकता है, लेकिन फ़ाइल आकार के साथ मेमोरी उपयोग बढ़ता है। बहुत बड़े फ़ाइलों के लिए पृष्ठों को व्यक्तिगत रूप से प्रोसेस करने पर विचार करें।

**प्रश्न: मैं Aspose.Page के लिए अस्थायी लाइसेंस कैसे प्राप्त करूँ?**  
**उत्तर:** आप अस्थायी लाइसेंस **[यहाँ](https://purchase.aspose.com/temporary-license/)** से प्राप्त कर सकते हैं।

**प्रश्न: सहायता या Aspose समुदाय से जुड़ने के लिए कहाँ जाऊँ?**  
**उत्तर:** प्रश्न पूछने और अनुभव साझा करने के लिए **[Aspose.Page फ़ोरम](https://forum.aspose.com/c/page/39)** पर जाएँ।

## निष्कर्ष

इस ट्यूटोरियल में हमने Aspose.Page for .NET का उपयोग करके **XPS दस्तावेज़ फ़ाइलों को संशोधित** करने और कस्टम हस्ताक्षर टेक्स्ट जोड़ने का प्रदर्शन किया। अब आपके पास विशिष्ट पृष्ठों पर कोई भी टेक्स्ट, वॉटरमार्क या एनोटेशन डालने की ठोस नींव है। विभिन्न फ़ॉन्ट, रंग और स्थितियों के साथ प्रयोग करें ताकि आपके एप्लिकेशन की ब्रांडिंग आवश्यकताओं को पूरा किया जा सके, और उन्नत ग्राफ़िक्स व लेआउट क्षमताओं के लिए व्यापक Aspose.Page API का अन्वेषण करें।
