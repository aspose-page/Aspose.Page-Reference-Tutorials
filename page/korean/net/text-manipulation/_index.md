# Aspose.Page 텍스트 조작

## 소개

.NET용 **aspose.page 텍스트 조작**의 세계에 오신 것을 환영합니다. 이 포괄적인 가이드에서는 PostScript(PS) 및 XPS 문서 내부에 텍스트를 추가, 편집 및 스타일링하는 데 필요한 핵심 기술을 단계별로 안내합니다. 보고 엔진을 구축하거나, 인쇄 가능한 청구서를 생성하거나, 맞춤형 그래픽을 만들고자 할 때, 이러한 기능을 마스터하면 .NET 프로젝트가 보다 강력하고 유연해집니다.

## 빠른 답변
- **무엇을 조작할 수 있나요?** PostScript와 XPS 문서 모두에 일반 텍스트 또는 유니코드 텍스트를 추가할 수 있습니다.  
- **필요한 라이브러리는?** Aspose.Page for .NET이 전체 텍스트‑조작 API 세트를 제공합니다.  
- **라이선스가 필요합니까?** 개발 단계에서는 무료 체험판으로 충분하지만, 운영 환경에서는 상용 라이선스가 필요합니다.  
- **지원되는 플랫폼?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.  
- **소요 시간은?** 간단한 텍스트 삽입은 10분 이내에 구현할 수 있습니다.

## Aspose.Page 텍스트 조작이란?
Aspose.Page의 텍스트 조작 기능을 사용하면 문자열을 프로그래밍 방식으로 배치하고, 글꼴을 제어하며, 유니코드 문자를 적용하고, 페이지에 텍스트를 정확히 위치시킬 수 있습니다. API는 저수준 PS/XPS 구문을 추상화하므로 문서 내부 구조보다 비즈니스 로직에 집중할 수 있습니다.

## 왜 Aspose.Page를 텍스트 조작에 사용해야 할까요?
- **Cross‑format support** – 한 번 작성으로 PostScript와 XPS 모두에 출력합니다.  
- **Full Unicode handling** – 국제 문서에서 누락된 문자가 더 이상 발생하지 않습니다.  
- **No external dependencies** – 순수 .NET 라이브러리이며 네이티브 바이너리가 필요 없습니다.  
- **High performance** – 대규모 문서 생성에 최적화되었습니다.

## 사전 요구 사항
- Visual Studio 2022 또는 .NET 호환 IDE.  
- .NET 6 SDK (또는 .NET Framework 4.5+).  
- Aspose.Page for .NET NuGet 패키지 설치 (`Install-Package Aspose.Page`).

## 자주 묻는 질문

**Q:** *이 튜토리얼을 상용 애플리케이션에서 사용할 수 있나요?*  
**A:** 예. 상용 환경에서는 유효한 Aspose.Page 라이선스가 필요합니다; 평가용으로는 무료 체험판을 사용할 수 있습니다.

**Q:** *Aspose.Page가 오른쪽에서 왼쪽으로 쓰는 언어를 지원하나요?*  
**A:** 물론입니다. 유니코드 처리에 RTL 스크립트(예: 아랍어, 히브리어)도 포함됩니다.

**Q:** *기존 PS/XPS 파일에 텍스트를 추가하려면 어떻게 해야 하나요?*  
**A:** API를 사용해 기존 문서를 로드하고 페이지를 수정한 뒤 전체 파일을 다시 생성하지 않고 결과를 저장할 수 있습니다.

**Q:** *텍스트를 배치하기 전에 크기를 측정할 방법이 있나요?*  
**A:** 예. 선택한 폰트를 기준으로 `TextFragment` 측정 메서드를 사용해 너비와 높이를 계산할 수 있습니다.

**Q:** *폰트를 직접 관리해야 하나요?*  
**A:** Aspose.Page는 시스템 폰트를 자동으로 임베드하지만, 플랫폼 간 일관된 렌더링을 위해 사용자 지정 폰트 파일을 제공할 수도 있습니다.
