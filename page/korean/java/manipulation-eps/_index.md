## Common Issues & Tips

- **Missing fonts** – EPS에 참조된 폰트가 호스트 머신에 설치되어 있는지 확인하거나 API를 사용해 임베드하세요.  
- **Large file size after resize** – `saveOptions.setCompressionLevel()` 메서드를 사용해 품질 손실 없이 파일 크기를 줄이세요.  
- **Unexpected clipping** – 자르기 사각형 좌표를 다시 확인하세요; 좌표는 포인트(1 pt = 1/72 인치) 단위로 표현됩니다.  

## Frequently Asked Questions

**Q: 여러 EPS 파일을 배치 처리할 수 있나요?**  
A: 예. 자르기 또는 크기 조정 로직을 루프 안에 넣고 각 파일마다 동일한 `Page` 객체를 재사용하면 됩니다.

**Q: Aspose.Page가 임베드된 래스터 이미지가 포함된 EPS를 지원하나요?**  
A: 물론입니다. 벡터 요소와 래스터 요소 모두 조작 중에 보존됩니다.

**Q: 편집 후 EPS를 다른 형식(PDF, PNG)으로 변환할 수 있나요?**  
A: 예. 자르기 또는 크기 조정 후 `page.save("output.pdf", SaveFormat.Pdf)` 또는 `SaveFormat.Png`을 호출하면 됩니다.

**Q: EPS 파일이 손상된 경우 어떻게 해야 하나요?**  
A: API가 `FileFormatException`을 발생시킵니다. 예외를 잡아 처리하고, 소스 파일을 검증한 후 진행하세요.

**Q: 리소스를 수동으로 닫아야 하나요?**  
A: `Page` 클래스는 `Closeable`을 구현합니다; try‑with‑resources 블록을 사용하거나 `page.close()`를 호출해 메모리를 해제하세요.
