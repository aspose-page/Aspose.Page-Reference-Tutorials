# PostScript에 텍스트 추가하기 (Aspose.Page for Java 사용)

## 소개

이 튜토리얼에서는 Aspose.Page for Java를 사용하여 PostScript 문서에 **텍스트 추가 방법**을 알아봅니다. 간단한 라벨, 복잡한 다국어 레이아웃, 혹은 맞춤형 스타일 헤딩이 필요하든, 이 가이드는 모든 단계를 안내합니다. 먼저 일반 텍스트 삽입의 기본을 살펴보고, 이후 Unicode 문자열과 맞춤 폰트 처리 방법을 탐구하여 진정으로 동적인 PostScript 파일을 만들 수 있도록 합니다.

## 빠른 답변
- **주요 목표는 무엇인가요?** PostScript 파일에 프로그래밍 방식으로 텍스트를 추가합니다.  
- **사용된 라이브러리는?** Aspose.Page for Java.  
- **라이선스가 필요합니까?** 평가를 위해 무료 체험판을 사용할 수 있으며, 실제 운영을 위해서는 상업용 라이선스가 필요합니다.  
- **Unicode 문자를 사용할 수 있나요?** 예 – API는 Unicode 문자열을 완전히 지원합니다.  
- **필요한 Java 버전은?** Java 8 이상.

## PostScript에서 텍스트 조작이란?

텍스트 조작이란 PostScript 페이지 안에 문자들을 배치하고, 스타일을 지정하며, 렌더링하는 과정을 말합니다. Aspose.Page를 사용하면 저수준 PostScript 구문을 직접 다루지 않고도 폰트 선택, 위치 지정, 인코딩을 제어할 수 있습니다.

## 왜 Aspose.Page로 PostScript에 텍스트를 추가하나요?

- **크로스 플랫폼 일관성:** PostScript를 지원하는 모든 시스템에서 동일한 출력을 생성합니다.  
- **전체 Unicode 지원:** 추가 인코딩 없이 다국어 문서를 만들 수 있습니다.  
- **맞춤 폰트 통합:** 시스템 폰트와 임베디드 폰트를 모두 사용하여 브랜드 일관성을 유지합니다.  
- **프로그래밍 제어:** Java 코드에서 직접 보고서 생성, 청구서, 그래픽 등을 자동화합니다.

## Common Pitfalls & Tips

- **폰트를 찾을 수 없음:** 폰트 파일이 JVM에서 접근 가능하도록 하거나 `FontRepository`를 사용해 폰트를 임베드하세요.  
- **잘못된 인코딩:** 비ASCII 문자를 사용할 때는 항상 `UnicodeString`을 사용해 깨진 출력을 방지하세요.  
- **위치 지정 문제:** PostScript는 좌하단을 원점으로 사용하므로 Y 좌표를 적절히 조정하세요.

## Frequently Asked Questions

**Q: 맞춤 TrueType 폰트를 PostScript 파일에 임베드할 수 있나요?**  
A: 예. 텍스트 객체를 만들기 전에 `FontRepository.addFont("path/to/font.ttf")`를 사용하세요.

**Q: 텍스트를 회전시킬 수 있나요?**  
A: 물론입니다. `TextState.setRotation()` 메서드를 통해 텍스트 회전 각도를 설정하세요.

**Q: 문서 스트림을 수동으로 닫아야 하나요?**  
A: `Document.save()` 메서드가 스트림 종료를 처리하지만, 열어둔 맞춤 스트림은 직접 해제하는 것이 좋습니다.

**Q: Aspose.Page는 오른쪽에서 왼쪽으로 쓰는 언어를 어떻게 처리하나요?**  
A: 적절한 스크립트를 포함한 Unicode 문자열을 제공하고 `TextState`에서 텍스트 방향을 설정하면 됩니다.

**Q: 대용량 PostScript 파일의 성능 고려사항은 무엇인가요?**  
A: 폰트를 한 번만 로드하고 `TextState` 객체를 재사용하며 불필요한 임시 페이지 생성을 피하세요.
