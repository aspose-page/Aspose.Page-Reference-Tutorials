# Πώς να Συγχωνεύσετε Έγγραφα XPS με το Aspose.Page για .NET

## Εισαγωγή

Αναζητάτε έναν αξιόπιστο τρόπο **πώς να συγχωνεύσετε αρχεία XPS** προγραμματιστικά; Σε αυτό το tutorial θα σας καθοδηγήσουμε βήμα προς βήμα στη συγχώνευση εγγράφων XPS χρησιμοποιώντας το Aspose.Page για .NET. Είτε χρειάζεστε να συνδυάσετε αναφορές, τιμολόγια ή οποιαδήποτε άλλα περιουσιακά στοιχεία βασισμένα σε XPS, η διαδικασία είναι απλή και πλήρως αυτοματοποιημένη. Ας βουτήξουμε και δούμε πώς μπορείτε να επιτύχετε ένα καθαρό, συγχωνευμένο αρχείο XPS με λίγες μόνο γραμμές κώδικα C#.

## Γρήγορες Απαντήσεις
- **Ποια βιβλιοθήκη διαχειρίζεται τη συγχώνευση XPS;** Aspose.Page for .NET  
- **Πόσο διαρκεί η υλοποίηση;** Συνήθως λιγότερο από 10 λεπτά  
- **Χρειάζομαι άδεια;** Απαιτείται άδεια για παραγωγική χρήση· διατίθεται δωρεάν δοκιμή  
- **Υποστηριζόμενες εκδόσεις .NET;** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7  
- **Μπορώ να συγχωνεύσω κρυπτογραφημένα αρχεία XPS;** Ναι – το Aspose.Page μπορεί να διαχειριστεί κρυπτογραφημένα έγγραφα  

## Τι είναι η Συγχώνευση Εγγράφων XPS;
Το XPS (XML Paper Specification) είναι μια μορφή εγγράφου σταθερής διάταξης που δημιουργήθηκε από τη Microsoft. Η συγχώνευση αρχείων XPS σημαίνει η συνένωση πολλαπλών εγγράφων XPS σε ένα ενιαίο, συνεχόμενο αρχείο, διατηρώντας την αρχική διάταξη, τις γραμματοσειρές και τα γραφικά.

## Γιατί να Χρησιμοποιήσετε το Aspose.Page για .NET;
- **Πλήρης έλεγχος** της διαδικασίας συγχώνευσης χωρίς την ανάγκη του Microsoft XPS Viewer  
- **Χωρίς εξωτερικές εξαρτήσεις** – όλα εκτελούνται μέσα στην εφαρμογή .NET  
- **Υψηλή απόδοση** – λειτουργεί αποδοτικά ακόμη και με μεγάλα έγγραφα  
- **Διαπλατφορμική** – συμβατό με .NET Framework, .NET Core και .NET 5+  

## Προαπαιτούμενα

Πριν ξεκινήσετε, βεβαιωθείτε ότι έχετε:

- Βασική κατανόηση της C# και του οικοσυστήματος .NET.  
- **Aspose.Page for .NET** εγκατεστημένο – μπορείτε να το κατεβάσετε [εδώ](https://releases.aspose.com/page/net/).  
- Ένα ή περισσότερα αρχεία XPS που θέλετε να συνδυάσετε.

## Εισαγωγή Ονομάτων Χώρων

Στο έργο C# σας, εισάγετε το όνομα χώρου που σας δίνει πρόσβαση στο API του XPS:

```csharp
using Aspose.Page.XPS;
```

## Βήμα 1: Ρύθμιση του Έργου Σας

Δημιουργήστε ένα νέο έργο C# console ή βιβλιοθήκης στο Visual Studio, Rider ή το προτιμώμενο IDE σας. Προσθέστε μια αναφορά στο Aspose.Page DLL (ή εγκαταστήστε το πακέτο NuGet `Aspose.Page`). Αυτό σας δίνει πρόσβαση στην κλάση `XpsDocument` που θα χρησιμοποιηθεί αργότερα.

## Βήμα 2: Αρχικοποίηση Ροών

Ανοίξτε τα πηγαία αρχεία XPS ως ροές εισόδου και δημιουργήστε μια ροή εξόδου για το συγχωνευμένο έγγραφο. Οι δηλώσεις `using` εξασφαλίζουν ότι όλες οι ροές κλείνουν σωστά μετά τη λειτουργία.

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize XPS output stream
using (System.IO.Stream outStream = System.IO.File.Open(dataDir + "mergedXPSfiles.xps", System.IO.FileMode.OpenOrCreate, System.IO.FileAccess.Write))
// Initialize XPS input stream
using (System.IO.Stream inStream = System.IO.File.Open(dataDir + "input.xps", System.IO.FileMode.Open))
```

## Βήμα 3: Φόρτωση Εγγράφου XPS

Δημιουργήστε ένα αντικείμενο `XpsDocument` από την κύρια ροή εισόδου. Το αντικείμενο `XpsLoadOptions` σας επιτρέπει να προσαρμόσετε τη συμπεριφορά φόρτωσης εάν χρειάζεται.

```csharp
XpsDocument document = new XpsDocument(inStream, new XpsLoadOptions());
```

## Βήμα 4: Δημιουργία Πίνακα Αρχείων XPS

Προετοιμάστε έναν πίνακα συμβολοσειρών που καταγράφει κάθε αρχείο XPS που θέλετε να συγχωνεύσετε. Η σειρά του πίνακα καθορίζει τη σειρά στο τελικό έγγραφο.

```csharp
string[] filesToMerge = new string[] { dataDir + "Demo.xps", dataDir + "sample.xps" };
```

## Βήμα 5: Συγχώνευση Αρχείων XPS

Καλέστε τη μέθοδο `Merge`, περνώντας τον πίνακα διαδρομών αρχείων και τη ροή εξόδου. Το Aspose.Page διαχειρίζεται όλη τη βαριά δουλειά—συνδυάζει τις σελίδες, διατηρεί τους πόρους και γράφει το τελικό αρχείο XPS.

```csharp
document.Merge(filesToMerge, outStream);
```

## Κοινά Προβλήματα & Συμβουλές

- **Αρχείο δεν βρέθηκε** – Ελέγξτε ξανά τις διαδρομές στο `filesToMerge`. Η χρήση του `Path.Combine` μπορεί να βοηθήσει στην αποφυγή λαθών διαχωριστικού διαδρομής.  
- **Χρήση μνήμης** – Όταν συγχωνεύετε μεγάλο αριθμό αρχείων, σκεφτείτε την επεξεργασία τους σε παρτίδες για να διατηρήσετε τη χρήση μνήμης χαμηλή.  
- **Κρυπτογραφημένα έγγραφα** – Εάν κάποιο πηγαίο XPS είναι προστατευμένο με κωδικό, φορτώστε το με τα κατάλληλα διαπιστευτήρια πριν τη συγχώνευση.  

## Συχνές Ερωτήσεις

**Ε1: Μπορώ να συγχωνεύσω αρχεία XPS διαφορετικών μεγεθών σελίδας;**  
Α: Ναι. Το Aspose.Page κανονικοποιεί αυτόματα τις διαστάσεις των σελίδων κατά τη συγχώνευση.

**Ε2: Υπάρχει όριο στον αριθμό των αρχείων XPS που μπορώ να συνδυάσω;**  
Α: Δεν υπάρχει σκληρό όριο, αλλά πολύ μεγάλες συλλογές μπορεί να επηρεάσουν την απόδοση· παρακολουθήστε τη χρήση μνήμης.

**Ε3: Χρειάζομαι ειδική άδεια για τη συγχώνευση κρυπτογραφημένων εγγράφων XPS;**  
Α: Απαιτείται πλήρης άδεια Aspose.Page για οποιοδήποτε χαρακτηριστικό παραγωγικού επιπέδου, συμπεριλαμβανομένης της διαχείρισης κρυπτογραφημένων εγγράφων.

**Ε4: Πώς μπορώ να προσθέσω προσαρμοσμένο υποσέλιδο σε κάθε σελίδα μετά τη συγχώνευση;**  
Α: Μετά τη συγχώνευση, μπορείτε να ανοίξετε ξανά το παραγόμενο XPS με `XpsDocument` και να χρησιμοποιήσετε το API σχεδίασης για να εισάγετε υποσέλιδα.

**Ε5: Υποστηρίζει το Aspose.Page το .NET Core;**  
Α: Απόλυτα. Η βιβλιοθήκη είναι συμβατή με .NET Core 3.1 και νεότερες εκδόσεις, καθώς και με .NET 5/6/7.
