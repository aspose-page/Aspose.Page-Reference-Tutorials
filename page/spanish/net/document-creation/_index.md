## ¿Qué es un documento XPS y por qué crear uno?

XPS (XML Paper Specification) es un formato de documento de diseño fijo desarrollado por Microsoft. Conserva el diseño, las fuentes y los gráficos en todas las plataformas, lo que lo hace ideal para facturas, informes y formularios imprimibles. Crear documentos XPS de forma programática te brinda control total sobre el diseño y la automatización, eliminando la autoría manual.

## Preguntas frecuentes

**Q: ¿Cómo inicio un nuevo documento XPS desde cero?**  
A: Instanciar la clase `Document`, agregar una `Page`, luego usar objetos `Graphics` para dibujar texto, imágenes o formas.

**Q: ¿Puedo convertir un PDF existente a XPS usando Aspose.Page?**  
A: La conversión directa de PDF a XPS la maneja Aspose.PDF, pero puedes exportar el contenido del PDF a imágenes e incrustarlas en un documento XPS con Aspose.Page.

**Q: ¿Es posible editar un archivo XPS existente sin recrearlo?**  
A: Sí – cargue el archivo con `Document.Load`, modifique páginas o añada nuevo contenido, luego guárdelo nuevamente.

**Q: ¿Cuál es la mejor manera de generar un archivo PostScript para impresión?**  
A: Use la misma API `Document`, pero llame a `Save` con la opción `SaveFormat.PostScript`. Esto garantiza una salida lista para impresora.

**Q: ¿Existen límites de tamaño para archivos XPS o PostScript?**  
A: La biblioteca maneja archivos grandes de manera eficiente; sin embargo, para documentos extremadamente extensos, considere transmitir el contenido y usar `Document.OptimizeResources()`.
