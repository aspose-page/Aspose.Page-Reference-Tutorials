## Häufige Fallstricke & Tipps

- **Bildgröße ist wichtig** – Große Bilder erhöhen den Speicherverbrauch. Vor dem Hinzufügen verkleinern oder komprimieren.
- **Koordinatensystem** – XPS verwendet 96 DPI; denken Sie daran, Pixelwerte beim Positionieren in Punkte umzuwandeln.
- **Transparenz** – PNG‑Bilder behalten Alpha‑Kanäle bei, stellen Sie jedoch sicher, dass der Ziel‑Viewer Transparenz unterstützt.
- **Platzierung debuggen** – Verwenden Sie `saveAsPdf()`, um eine schnelle PDF‑Vorschau zu erzeugen und die Koordinaten zu überprüfen.

## Häufig gestellte Fragen

**Q: Kann ich Bilder zu einer bestehenden XPS‑Datei hinzufügen?**  
A: Ja. Öffnen Sie die bestehende XPS mit `XpsDocument` und verwenden Sie die gleichen `addImage`‑APIs, um neue Grafiken einzufügen.

**Q: Welche Bildgrößenbeschränkungen gelten?**  
A: Aspose.Page unterstützt Bilder bis zu 10 MB; größere Dateien können die Leistung beeinträchtigen, sind aber weiterhin verarbeitbar.

**Q: Beeinflusst das Kacheln die Dateigröße?**  
A: Kacheln wiederholt dieselben Bilddaten, sodass die Dateigröße nur minimal wächst – es wird nur die Kacheldefinition gespeichert.

**Q: Gibt es eine Möglichkeit, die Bildtransparenz beizubehalten?**  
A: PNG‑Bilder behalten beim Hinzufügen die Alpha‑Kanäle, sodass transparente Bereiche im XPS korrekt gerendert werden.

**Q: Wie kann ich Platzierungsprobleme von Bildern debuggen?**  
A: Verwenden Sie die Methode `saveAsPdf`, um ein Vorschau‑PDF zu exportieren; es hilft, Koordinaten und Skalierung vor dem endgültigen XPS‑Export zu visualisieren.

**Q: Was ist, wenn ich ein Bild drehen oder skalieren muss?**  
A: Wenden Sie vor dem Aufruf von `addImage` eine Transformationsmatrix über das `Graphics`‑Objekt an.

**Q: Kann ich mehrere Bilder auf einer einzelnen Seite kombinieren?**  
A: Natürlich. Rufen Sie `addImage` mehrfach mit unterschiedlichen Rechtecken oder Transformations‑Einstellungen auf.
