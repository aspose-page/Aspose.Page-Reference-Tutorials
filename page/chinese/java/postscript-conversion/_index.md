## 常见问题及解决方案

| 问题 | 解决方案 |
|------|----------|
| **低分辨率图像** | 在调用 `save()` 之前，提高 `ImageSaveOptions` 的 DPI。 |
| **大文件的内存消耗** | 一次处理一页，并在保存后释放每个 `Image`。 |
| **受密码保护的 PostScript** | 使用接受密码字符串的 `LoadOptions` 构造函数。 |
| **颜色异常** | 检查 `PdfSaveOptions` 或 `ImageSaveOptions` 中的色彩空间设置。 |

## 常见问题

**Q: 我可以在商业项目中使用 Aspose.Page 转换吗？**  
A: 是的。生产使用需要有效的 Aspose 许可证，但可提供免费试用进行评估。

**Q: 支持哪些 Java 版本？**  
A: Aspose.Page for Java 支持 Java 8 及更高版本。

**Q: 如何控制导出图像的分辨率？**  
A: 在调用 `save` 方法之前，在 `ImageSaveOptions` 对象上设置 DPI 值。

**Q: 是否可以一次批量转换多个 PostScript 文件？**  
A: 当然可以——遍历文件集合，对每个文档应用相同的转换逻辑。

**Q: 如果我的 PostScript 文件受密码保护怎么办？**  
A: 使用接受 `LoadOptions` 对象的 `Document` 构造函数，在其中指定密码。

**Q: 如何减小生成的 PDF 大小？**  
A: 在 `PdfSaveOptions` 中启用压缩，并在不需要高分辨率时考虑对图像进行降采样。
