# รวมเอกสาร XPS ด้วย Aspose.Page สำหรับ .NET

## บทนำ

ในบทแนะนำนี้คุณจะได้เรียนรู้วิธี **รวมเอกสาร XPS** และจัดการหน้าของพวกมันโดยใช้ไลบรารี Aspose.Page ในสภาพแวดล้อม .NET ไม่ว่าคุณจะต้องการรวมหลายรายงานเป็นไฟล์ XPS เดียวหรือจัดเรียงหน้าเพื่อให้ได้ผลลัพธ์ที่เรียบร้อย คู่มือนี้จะพาคุณผ่านขั้นตอนต่าง ๆ อย่างเป็นขั้นเป็นตอน พร้อมคำอธิบายที่ชัดเจนและโค้ดที่พร้อมรัน

## คำตอบด่วน
- **Aspose.Page สามารถทำอะไรได้บ้าง?** รวมเอกสาร XPS, แทรก, เพิ่ม หรือ ลบหน้า, และบันทึกผลลัพธ์.  
- **ต้องการไลเซนส์สำหรับการทดสอบหรือไม่?** มีไลเซนส์ชั่วคราวสำหรับการประเมินผล.  
- **เวอร์ชัน .NET ที่รองรับคืออะไร?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **ต้องใช้ Visual Studio หรือไม่?** ไม่จำเป็น, IDE ใดก็ได้ที่รองรับ C# ก็ทำงานได้, แต่แนะนำให้ใช้ Visual Studio.  
- **กระบวนการรวมใช้เวลานานเท่าไหร่?** โดยทั่วไปใช้เวลาไม่กี่วินาทีสำหรับไฟล์ XPS ขนาดมาตรฐาน.

## การรวมเอกสาร XPS คืออะไร?
การรวมเอกสาร XPS หมายถึงการนำหน้าจากไฟล์ XPS ที่มีอยู่สองไฟล์หรือมากกว่า มารวมเป็นเอกสาร XPS เดียว นี่เป็นประโยชน์สำหรับการสร้างรายงานสรุป, รวบรวมคู่มือหลายหน้า, หรือเตรียมแพคเกจพร้อมพิมพ์โดยไม่ต้องแปลงเป็นรูปแบบอื่น.

## ทำไมต้องใช้ Aspose.Page สำหรับ .NET?
Aspose.Page มี **API .NET แท้** ที่ทำงานโดยตรงกับไฟล์ XPS — ไม่ต้องพึ่งเครื่องมือภายนอกหรือคอมโพเนนต์ของบุคคลที่สาม ให้คุณควบคุมลำดับหน้า, จุดแทรก, และการรักษาเนื้อหาได้อย่างละเอียด ทำให้กระบวนการรวมเป็นไปอย่างเชื่อถือได้และรวดเร็ว.

## ข้อกำหนดเบื้องต้น
- **Aspose.Page for .NET** – ดาวน์โหลดจาก [Aspose.Page for .NET documentation](https://reference.aspose.com/page/net/).  
- **สภาพแวดล้อมการพัฒนา** – Visual Studio, Rider, หรือ IDE ใดก็ได้ที่รองรับ C#.  
- **ไฟล์ XPS อินพุต** – ตัวอย่างไฟล์สามไฟล์ (`input1.xps`, `input2.xps`, `input3.xps`) ที่จัดเก็บในโฟลเดอร์ที่ทราบ.

## นำเข้า Namespaces

```csharp
using Aspose.Page.XPS;
using Aspose.Page.XPS.XpsModel;
using System.Drawing;
```

Namespaces เหล่านี้ให้คุณเข้าถึงคลาสเอกสาร XPS หลัก, โมเดลหน้า, และยูทิลิตี้การวาดพื้นฐาน.

## ขั้นตอนที่ 1: ตั้งค่าไดเรกทอรีเอกสาร

```csharp
string dataDir = "Your Document Directory";
```

แทนที่ **Your Document Directory** ด้วยเส้นทางเต็มที่ไฟล์ XPS ของคุณถูกเก็บไว้, ตัวอย่างเช่น `C:\\Docs\\XpsFiles\\`.

## ขั้นตอนที่ 2: สร้างอินสแตนซ์ XPS Document

```csharp
XpsDocument doc1 = new XpsDocument(dataDir + "input1.xps");
XpsDocument doc2 = new XpsDocument(dataDir + "input2.xps");
XpsDocument doc3 = new XpsDocument(dataDir + "input3.xps");
XpsDocument doc4 = new XpsDocument();
```

- `doc1`, `doc2`, และ `doc3` แสดงถึงเอกสารต้นทางที่คุณต้องการรวม.  
- `doc4` เป็นเอกสาร XPS ว่างที่จะเก็บผลลัพธ์ที่รวม.

## ขั้นตอนที่ 3: แทรก, เพิ่ม, และลบหน้า

```csharp
doc4.InsertPage(1, doc2.Page, false);
doc4.AddPage(doc3.Page, false);
doc4.RemovePageAt(2);
doc4.InsertPage(2, doc1.SelectActivePage(3), false);
```

นี่คือสิ่งที่แต่ละบรรทัดทำ:

1. **InsertPage(1, doc2.Page, false)** – วางหน้าที่หนึ่งของ `doc2` ที่ตำแหน่ง 1 ใน `doc4`.  
2. **AddPage(doc3.Page, false)** – เพิ่มหน้าที่หนึ่งของ `doc3` ไปยังส่วนท้ายของ `doc4`.  
3. **RemovePageAt(2)** – ลบหน้าที่อยู่ที่ดัชนี 2 (ใช้เพื่อลบหน้าที่ไม่ต้องการ).  
4. **InsertPage(2, doc1.SelectActivePage(3), false)** – แทรกหน้าที่สามของ `doc1` ไปยังตำแหน่ง 2, ทำให้การรวมเสร็จสมบูรณ์.

การดำเนินการเหล่านี้แสดงให้เห็นว่าคุณสามารถ **รวมเอกสาร XPS** พร้อมกับจัดเรียงลำดับหรือทิ้งหน้าออกตามที่ต้องการ.

## ขั้นตอนที่ 4: บันทึกเอกสารที่รวม

```csharp
doc4.Save(dataDir + "out.xps");
```

ไฟล์ XPS ที่รวมเสร็จ (`out.xps`) จะถูกเขียนไปยังไดเรกทอรีเดียวกัน คุณสามารถเปิดไฟล์นี้ด้วยโปรแกรมดู XPS ใดก็ได้หรือดำเนินการต่อด้วย Aspose.Page.

## ปัญหาทั่วไปและวิธีแก้
- **ไฟล์ไม่พบ** – ตรวจสอบเส้นทาง `dataDir` อีกครั้งและให้แน่ใจว่าไฟล์อินพุตมีอยู่.  
- **ดัชนีหน้าที่ไม่ถูกต้อง** – ดัชนีหน้านับจาก 1; การพยายามแทรกหน้าที่ไม่มีอยู่จะทำให้เกิดข้อยกเว้น.  
- **ข้อผิดพลาดไลเซนส์** – ใช้ไลเซนส์ชั่วคราวหรือเต็มก่อนการนำไปใช้งานจริง.

## คำถามที่พบบ่อย

### Q1: ฉันสามารถจัดการหน้าจากเอกสาร XPS ต่าง ๆ ได้หรือไม่?
A1: ได้, ตามที่แสดงในบทแนะนำ, คุณสามารถแทรกหน้าจากหลายเอกสาร XPS ไปยังเอกสารใหม่ได้.

### Q2: ฉันจะลบหน้าที่เฉพาะจากเอกสารได้อย่างไร?
A2: ใช้เมธอด `RemovePageAt` โดยระบุดัชนีของหน้าที่ต้องการลบ.

### Q3: Aspose.Page รองรับ Visual Studio หรือไม่?
A3: ใช่, Aspose.Page เข้ากันได้อย่างเต็มที่กับ Visual Studio ทำให้การรวมเข้ากับโครงการ .NET ของคุณเป็นเรื่องง่าย.

### Q4: มีตัวเลือกไลเซนส์ใดบ้าง?
A4: มี, คุณสามารถสำรวจตัวเลือกไลเซนส์และรับไลเซนส์ชั่วคราวได้ [ที่นี่](https://purchase.aspose.com/temporary-license/).

### Q5: ฉันจะหาการสนับสนุนหรือถามคำถามได้จากที่ไหน?
A5: เยี่ยมชม [Aspose.Page forum](https://forum.aspose.com/c/page/39) เพื่อรับการสนับสนุนและมีส่วนร่วมกับชุมชน.

## คำถามที่พบบ่อยเพิ่มเติม

**Q: ฉันสามารถรวมไฟล์ XPS มากกว่าสามไฟล์ได้หรือไม่?**  
A: แน่นอน. สร้างอินสแตนซ์ `XpsDocument` เพิ่มเติมและใช้ `InsertPage` หรือ `AddPage` อย่างต่อเนื่องเพื่อสร้างเอกสารที่รวมใหญ่ขึ้น.

**Q: การรวมรักษาการจัดรูปแบบและกราฟิกเดิมหรือไม่?**  
A: ใช่. Aspose.Page คัดลอกเนื้อหาหน้าแบบไบต์ต่อไบต์ ดังนั้นข้อความ, รูปภาพ, และกราฟิกเวกเตอร์จะคงเดิม.

**Q: ฉันจะแทรกหน้าที่ส่วนท้ายโดยไม่ระบุตำแหน่งอย่างไร?**  
A: ใช้ `AddPage(sourcePage, false)` ซึ่งจะเพิ่มหน้าที่ส่วนท้ายของเอกสาร.

**Q: สามารถรวมเอกสาร XPS บนเซิร์ฟเวอร์โดยไม่มี UI ได้หรือไม่?**  
A: API ทำงานแบบไม่มี UI อย่างเต็มที่; คุณสามารถรันโค้ดเดียวกันใน ASP.NET, Azure Functions, หรือสภาพแวดล้อม .NET ฝั่งเซิร์ฟเวอร์ใดก็ได้.

**Q: หากไฟล์ XPS ของฉันถูกป้องกันด้วยรหัสผ่านจะทำอย่างไร?**  
A: Aspose.Page ปัจจุบันไม่รองรับไฟล์ XPS ที่เข้ารหัส; คุณต้องถอดรหัสไฟล์ก่อนทำการรวม.
